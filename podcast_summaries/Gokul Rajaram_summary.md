# Gokul Rajaram - 播客摘要

**嘉宾简介**: Gokul Rajaram - DoorDash产品/业务负责人，Coinbase/Pinterest/Trade Desk董事会成员，前Google/Facebook/Square/Caviar高管，著名天使投资人

## 核心议题 (The Questions)

1. **如何选择工作机会？** 什么样的公司和创始人值得加入
2. **如何在大型组织中设置产品开发流程？** 从25人到1000人的演进
3. **如何招聘第一个PM？** 何时招聘、从哪里找、如何评估
4. **如何成为有效的天使投资人？** Founder-centric投资哲学

## 核心洞察 (Core Conclusions)

1. **意外的Google AdSense故事**：Gokul最初在Google做syndication（将搜索结果和广告分发到AOL、Yahoo等）。下午6点后，他发现一组工程师在"反向Google搜索"——从网页提取关键词以定位广告。他成为"兼职PM"，3个月内这个项目成长为Google AdSense。
   - **教训**：做好核心工作的同时，保持好奇心，对机缘巧合开放。在Zoom时代这更难，但要了解公司在做什么，与人建立关系。

2. **"机缘巧合"的力量**：Gokul建议了一个公司（被Square收购），Square CEO问最好的产品人选是谁，他的名字被提到。他从未在那家公司有股权，只是"非正式建议"。
   - **教训**：向前看（Pay it forward）。你建立的好意储备会以不同方式回来帮助你。

3. **选择工作机会的三个标准**：
   - **创始人不为钱而来**：真正活体呼吸使命。如果他们不断谈论收入或让你变富，那不是建立大公司的创始人
   - **解决你获得能量的问题**：Gokul从服务小企业或消费者中获得能量（可以在街上见到他们），而不是大企业
   - **成为该领域的领导者**：在二级或三级电子商务公司做高级人员 vs 在Coinbase（领导者）做较低级别——总是选择领导者。Google vs Yahoo：Google的ICPM vs Yahoo的VP产品

4. **何时加入公司的建议**：
   - **新员工**：加入中期阶段公司（300-500人），已有产品市场适应和渠道适应（product-market-channel fit）
   - **职业中期**：除非你非常了解创始人，否则避免非常早期阶段（未来1-2年市场将很残酷）
   - **Gokul的路径**：Google的员工#6-800、Facebook的几百人、Square的700-800、DoorDash的1500人

5. **产品开发流程的演进**：
   - **早期**：每周计划（活死逐周）
   - **中期**：季度目标 + 每周计划
   - **大型**：年度计划 → 季度目标 → 每周计划
   - **工具**：开始用简单电子表格，不要使用复杂的项目管理工具
   - **战略分离**：早期产品战略 = 公司战略；后期产品战略独立

6. **招聘第一个PM的时机**：
   - **5-10个工程师**时需要一个专业PM
   - **除非**：创始人想扮演PM角色，或工程师真正理解领域（Stripe、基础设施公司、ML公司需要更少PM）
   - **最佳来源**：从内部转换（分析师、工程师、设计师）
   - **原因**：工程师和创始人信任他们，他们知道如何工作

7. **第一个PM失败的迹象**：
   - 工程师会告诉你（他们很直接）
   - 过多陷入"如何构建产品"（工程师/设计领域）
   - 需要教练帮助他们转向客户和问题解决

8. **产品经理汇报线**：
   - **3-5个PM**：应该向专业产品领导汇报（需要导师、教练、文化建设）
   - **原因**：一个坏PM可以搞砸10个工程师的工作；一个好的PM可以产生10倍价值
   - **该产品领导可以汇报给**：CTO、CEO或GM（不重要）

9. **"招聘领导者"的秘诀**：
   - **不要招聘该公司的负责人**，招聘他们的副手
   - **甚至副手的副手**
   - 构建组织图（LinkedIn、朋友）
   - **Square例子**：2015年上市后，支付/合规/风险团队被大量挖走，其他fintech没有去负责人，而是去了副手和副手的副手

10. **Founder-centric投资哲学**：
    - **市场很重要，但伟大的创始人创造自己的市场**（Airbnb例子）
    - 如果优秀的创始人在第一次尝试中失败，他们会利用学习在相同或相似空间开始另一家公司并成功
    - **Y Combinator模式**：投资于builder，他们可以快速转型

11. **Founder寻找标准**：
    - **真实原因**：他们为什么开始这个公司？他们观察过的问题 vs 在TechCrunch读到的
    - **两个人创始团队**：至少一个builder和一个seller（或hustler和hacker）
    - **在产品前雇佣人才**：如果你能展示你有一群了不起的人排队加入
    - **Collison兄弟例子**：买书/论文阅读以完全理解支付

12. **时间管理**：
    - **DoorDash**：每天10小时，最多2-3小时用于其他事情
    - **会议**：每天6个半小时会议（周一至周五30个）+ 周末每天8个（2天16个）= 总共46个会议/周
    - ** compartments**：会议后做笔记，然后立即忘记，进入下一个会议
    - **标准问题**："告诉我你的创立故事"

13. **头衔建议**：
    - **尽可能延迟**给予头衔
    - **避免**：Director和VP头衔（导致最多冲突、怨恨、hr头痛）
    - **使用**："Lead"或"Head of"（更清晰）
    - **Square**：没有关于头衔的讨论，只有范围和影响
    - **Jack Dorsey**：现在是"Block Head"

## 心法与原则

1. **机缘巧合（Serendipity）**：Lenny有一个TEDx演讲"Losing Your Serendipity"。Gokul说：保持好奇心，对机缘巧合开放，与人们建立关系。伟大的职业生涯不是通过在工作中晋升建立的，而是通过认识做伟大工作的人，他们知道并希望你加入他们的团队，然后等待机缘巧合并抓住它。

2. **向前看（Pay it Forward）**：Lenny是这方面的榜样。你建立的好意储备会以不同方式回来帮助你。

3. **Founder真实性**：每个创始人都有独特的风格，但他们不试图改变自己的风格。公司在某种程度上以他们的形象建立。如果Jack试图建立一家企业软件公司，我认为那不会奏效。那不是他。

4. **个人品牌**：当你用Google搜索自己时，如果你的LinkedIn档案是第一个出现的，你可能做错了。应该出现的是你写的文章、你发布的tweet、别人在别处提到你的东西。**没人能替你建立你的品牌**。你必须超越你工作的公司。

5. **"钓鱼的故事"哲学**：不要辛苦工作10年最终能够写作，而是**直接开始写作**。

## 实战手册

### 选择工作机会的框架：

**三个问题**：
1. 创始人真正活体呼吸使命吗？（不是为了钱）
2. 我会从这个**问题**和**客户**中获得能量吗？
3. 这个公司能成为其领域的领导者吗？

**市场阶段选择**：
- **新员工**：中期公司（300-500人），已有产品市场和渠道适应
- **有经验**：除非非常了解创始人，否则避免早期（市场将很残酷）

### 产品开发流程设置：

**早期（<25人）**：
- 每周计划
- 简单电子表格
- 产品战略 = 公司战略

**中期（25-100人）**：
- 季度目标 + 每周计划
- 开始写战略文档
- 产品战略从公司战略分离

**大型（100+人）**：
- 年度计划 → 季度目标 → 每周计划
- 多个互锁的产品战略文档
- 跨功能战略文档

**原子团队**：
- 6-7人（工程师、产品、设计、分析师）
- 每日站会
- 每周任务回顾

### 招聘第一个PM：

**时机**：5-10个工程师时（除非创始人想扮演PM角色）

**来源**：从内部转换（分析师、工程师、设计师）

**为什么内部**：
- 创始人信任他们
- 工程师信任他们
- 他们知道公司如何工作
- 器官排斥风险（从外部引入会失败）

**评估**：
- 好奇心、客户中心、沟通能力、促进讨论、解决问题
- 6个月可以成为好的PM（有导师）

**失败的迹象**：
- 工程师告诉你他们不工作
- 过多陷入"如何构建"（需要教练转向客户）

### 招聘领导者的"副手"策略：

**流程**：
1. 确定同类空间中最好的公司（非竞争对手）
2. 确定在该功能上最好的3-4个公司
3. 构建组织图（LinkedIn、朋友）
4. **不要招聘负责人**，招聘**副手**或**副手的副手**

**例子**：
- Square 2015年上市后成为唯一公开fintech
- 支付/合规/风险团队被大量挖走
- 其他公司没有去负责人，而是去了副手和副手的副手

**提示**：可以问"谁是你知道的最伟大的[功能]人才？"（不是"我想要谁"）

### Founder寻找标准：

**真实原因**：
- 他们观察过的问题吗？
- 他们已经深入那个行业了吗？
- Collison兄弟买书/论文理解支付

**团队组成**：
- **两个人**是理想的
- Builder + Seller（或Hustler + Hacker）
- 没有人拥有所有技能

**能力信号**：
- 在产品前雇佣人才
- 承诺加入或资助的人

### 天使投资建议：

**建立交易流的方法**：
1. **建立你的品牌**（写作、社交媒体）
2. **告诉其他天使投资人**你想要帮助公司
3. **VC**（通常有天使分配）

**差异化**：
- 你的独特价值是什么？
- 为什么公司应该接受你的钱？
- 通过分享非显而易见的见解来做到这一点

**Founder-centric vs Market-centric**：
- Market-centric：相信大市场
- Founder-centric（Gokul）：相信伟大的创始人（可以转型、创造新市场）
- YC模式：投资于builder，他们可以快速转型

### 时间管理策略：

**每日分配**：
- 主要工作（DoorDash）：7-8小时
- 其他事情：2-3小时（最多）

**会议容量**：
- 工作日：6个会议（30分钟）
- 周末：每天8个会议
- 总计：46个会议/周

**标准问题**：
- "告诉我你的创立故事"（创始人喜欢讲这个）
- 然后有一套标准问题，这样你可以花时间倾听而不是思考问题

**Compartments**：
- 会议后做笔记
- 然后立即忘记
- 进入下一个会议

### Founder日常时间分配建议：

**招聘**：
- 创始人应该每天花**2小时**招聘
- 1小时：触达（LinkedIn等）——可以是"busy work"时间
- 1小时：会见1-2人（两个半小时会议）

**计算**：
- 30天 = 60次会议
- 30小时LinkedIn（工作日20天，每天1.5小时）

### 头衔策略：

**Square模式**：
- Jack Dorsey看到Twitter头衔被武器化
- Square没有Director/Senior Director头衔
- 唯一关于头衔的讨论
- 只讨论范围、影响、薪酬

**推荐**：
- 延迟给予头衔
- 避免：Director、VP
- 使用："Lead"、"Head of"
- 好处：人们为范围和影响留下，而不是头衔

**例子**：
- Gokul在Square的头衔："Caviar Lead"
- 他团队的："Caviar产品Lead"、"Caviar工程Lead"、"Caviar战略和运营Lead"、"Caviar销售Lead"
- Jack Dorsey现在："Block Head"

## 反常识视角

1. **工作机会选择**：在二级或三级公司做高级人员 vs 在领导者公司做较低级别——总是选择领导者。Google的ICPM vs Yahoo的VP产品。

2. **第一个PM的来源**：不要从外部招聘"最好"的功能PM。从内部转换分析师/工程师/设计师。他们可能不是功能上最好的PM，但文化同化最好。

3. **招聘领导者**：不要招聘该公司的负责人，招聘**副手**或**副手的副手**。

4. **产品经理影响**：一个坏PM可以搞砸10个工程师的工作。一个好的PM可以产生10倍价值。

5. **头衔**：Director和VP头衔导致最多冲突、怨恨和hr头痛。避免它们。

6. **早期公司**：除非你非常了解创始人，否则避免加入（未来1-2年市场将很残酷）。

7. **天使投资**：你可能会错过"omission sins"（没投资于伟大创始人）。Gokul宁愿犯"commission sins"（投资于朋友失败），也不愿犯"omission sins"（没投资于朋友成功）。

8. **CAC vs LTV**：如果你早期没有LTV（Duolingo 3年没有货币化），CAC就没有意义。这迫使你专注于有机增长和保留率。
