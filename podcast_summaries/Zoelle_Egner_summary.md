# Zoelle Egner - 播客摘要

## 嘉宾简介

Zoelle Egner 是营销和客户成功领域的实战专家。她是 Airtable 的第 11 号员工，在公司早期同时领导营销和客户成功团队。在加入 Airtable 之前，她曾在创业公司工作，并在 COVID-19 期间共同领导了 VaccinateCA 非营利项目。Zoelle 在帮助初创公司"以小博大"（punching above their weight）方面拥有丰富经验，擅长在资源有限的情况下创造显著影响。

## 核心议题

本次播客深入探讨了**初创公司营销与客户成功**的实战策略，包括：
- 初创公司如何"以小博大"
- Airtable 早期的非常规营销策略
- 客户成功与营销的深度融合
- VaccinateCA 的危机营销经验
- 避免初创公司营销的常见陷阱

## 核心洞察

### 1. **"以小博大"（Punching Above Your Weight）的艺术**
> "小公司不应该试图模仿大公司的营销。相反，要做大公司不会做或不能做的事情。"

**核心策略**：

**不对称竞争**：
- **不要**：在 Google Ads 上与大公司竞价（他们会赢）
- **要**：找到大公司忽视的渠道或方法

**创意胜于预算**：
- 大公司依赖预算，小公司依赖创造力
- 创意传播速度更快、成本更低
- 示例：VaccinateCA 通过志愿者网络，两周内达到全美覆盖

**专注与深度**：
- 大公司必须迎合所有人
- 小公司可以深度服务于特定群体
- 深度服务创造热情的倡导者

### 2. **Airtable 的早期营销策略**

#### 策略一：比尔boards 的信号作用
> "我们没有期待从 billboard 获得直接客户。它的目的是向世界宣告：'Airtable 是真实存在的，我们正在成长'。"

**目的**：
- **招聘信号**：吸引顶尖人才注意到公司
- **存在证明**：向投资者和客户展示公司实力
- **品牌建设**：在目标受众心中建立认知

**教训**：
- 不是所有营销都需要直接 ROI
- 有时"信号"比转化更重要
- 早期品牌建设可以采用意想不到的方式

#### 策略二：Facebook 广告的精准定位
**目标受众**："Tinkerers"（摆弄者、探索者）

**为什么有效**：
- Airtable 是高度可定制的产品
- 摆弄者喜欢探索工具的可能性
- 这些人会成为产品热情的倡导者

**广告内容**：
- 展示实际的 Airtable 使用案例
- 不是"注册"而是"看你能创造什么"
- 强调创造力和可能性

#### 策略三：高端周边（AirPods）
> "我们送给早期用户的不是 T 恤，而是 AirPods。这创造了巨大的影响力和忠诚度。"

**为什么有效**：
- **差异化**：没人送 AirPods，这让 Airtable 难忘
- **实用性**：人们每天使用，持续提醒品牌
- **社交信号**：人们会问"你从哪里得到的？"
- **价值感知**：显示公司重视用户

**代价**：
- 昂贵（每件 ~$150-200）
- 只送给真正重要的早期用户
- 但创造的口碑价值远超成本

### 3. **客户成功 = 营销（Customer Success = Marketing）**
> "在早期公司，客户成功和营销不应该分开。它们是同一个功能的不同面向。"

**传统错误**：
- 营销负责获取客户
- 客户成功负责留住客户
- 两者之间没有沟通

**更好的方法**：

**信息流动**：
- 客户成功 → 营销："什么让客户成功？"
- 营销 → 客户成功："我们在承诺什么？"

**共同指标**：
- 不只是获取数量
- 而是获取"对"的客户
- 留存和扩张应该影响营销策略

**实际应用**：
- **成功案例**：客户成功团队识别最佳用户，营销团队找到更多这样的人
- **入职优化**：营销承诺的价值，客户成功团队确保兑现
- **反馈循环**：客户问题成为营销内容和定位改进的来源

### 4. **VaccinateCA 的危机营销经验**
> "我们在两周内建立了一个全美疫苗定位网络，依靠的不是预算，而是清晰的使命和志愿者网络。"

**关键学习**：

**清晰使命的威力**：
- 使命：帮助人们找到 COVID-19 疫苗
- 简单、紧迫、有意义
- 人们自愿加入，因为使命重要

**去中心化执行**：
- 中心团队提供工具和指导
- 各地志愿者自己收集信息
- 信任本地知识

**速度胜于完美**：
- 没有等待完美的系统
- 快速推出、快速迭代
- Google Sheets 作为起点

**传播策略**：
- 不依赖传统媒体
- 社交媒体和口碑
- 社区领袖和倡导者

**对初创公司的启示**：
- 清晰使命吸引资源
- 去中心化可以快速扩张
- 不要等待完美，开始并改进

### 5. **模板策略：客户成功而非 SEO**
> "Airtable 的模板库不是为了 SEO。它存在是因为模板是我们最好的客户成功工具。"

**传统观点**：
- 模板是获取搜索流量
- 人们在寻找"项目管理模板"时找到你

**Airtable 的发现**：
- **主要价值**：帮助用户理解如何使用产品
- **次要价值**：确实帮助 SEO，但不是主要目的

**策略转变**：
- 不是"什么搜索量高"
- 而是"什么帮助用户成功"
- 如果两者重叠，完美；如果不重叠，优先考虑用户成功

**教训**：
- 不要让 SEO 驱动产品决策
- 专注于真正帮助用户
- 价值自然会产生流量

### 6. **PR 和发布的正确方法**
> "大多数初创公司在 PR 上浪费太多时间。对于 90% 的公司，PR 应该是最小化的事。"

**常见错误**：
- 过早投入 PR
- 期望 PR 带来显著用户增长
- 在"发布"上花费太多精力

**更好的方法**：

**何时忽略 PR**：
- 产品市场匹配（PMF）之前
- 有更高杠杆的活动时
- PR 可能分散资源时

**何时做 PR**：
- 有真正新闻时（融资、重大产品发布）
- 可以借势时
- 已经有增长动能时

**如何做 PR**：
- **不要**雇佣昂贵的 PR 公司
- **要**直接与记者建立关系
- **不要**期待 TechCrunch 会改变你的业务
- **要**将 PR 视为品牌建设的补充，不是主要增长引擎

### 7. **避免"创造类别"陷阱**
> "投资者喜欢'创造类别'，但用户不关心。用户只想要解决他们的问题。"

**问题**：
- 创造新类别需要大量教育
- 用户不知道你在说什么
- 你花费时间解释类别，而不是价值

**更好的方法**：
- 使用用户已经理解的类别
- 专注于你做得更好/不同
- 让你的优势说话，而不是新词汇

**示例**：
- ❌ "我们是低代码无代码工作流自动化平台"
- ✅ "我们让任何人轻松创建自定义应用，就像 Excel 但强大得多"

**例外**：
- 如果类别真正不存在且用户已经在寻找
- 如果你有大量预算来教育市场
- 如果不创造类别无法解释价值

### 8. **早期营销团队招聘**
> "不要过早雇佣传统的营销人员。早期营销需要的是'瑞士军刀'型的人。"

**寻找的特征**：

**多面手（Generalist）**：
- 可以写文案、管理广告、做客户支持
- 不害怕做脏活
- 学习速度快

**数据驱动**：
- 理解指标
- 愿意实验
- 不依赖"直觉"

**产品理解**：
- 真正理解产品价值
- 能清楚传达价值主张
- 与产品团队良好合作

**避免**：
- 只擅长一个狭窄领域的专家
- 来自大公司的"我需要团队才能做 X"的人
- 只关注品牌建设而忽略增长的人

### 9. **社区驱动增长**
> "Airtable 的增长很多来自社区。人们创建模板、分享教程、帮助他人。"

**如何建立社区**：

**识别热情用户**：
- 他们自然帮助他人
- 他们分享自己的使用案例
- 他们提供反馈

**给予平台**：
- 让他们被看到（用户故事）
- 给他们早期访问功能
- 倾听他们的反馈

**不要过度管理**：
- 社区不应该感觉被控制
- 让它自然生长
- 提供资源，不要强制方向

**回报**：
- 免费的支持团队
- 真实的倡导者
- 持续的反馈来源

### 10. **Airtable 的 B2B2C 模式**
> "Airtable 最初是 B2C，但我们的 B2B 增长来自人们在个人使用后推荐给公司。"

**增长路径**：
1. 个人在工作中尝试 Airtable
2. 创建有用的应用
3. 与同事分享
4. 团队开始使用
5. 公司需要企业功能

**对营销的影响**：
- 不需要过早关注企业销售
- 让产品价值推动采用
- 企业功能响应需求，不是预先规划

**教训**：
- 最好的 B2B 营销可能是优秀的产品
- 让用户成为你的销售人员
- 企业功能应该在有需求时建设

## 心法与原则

### 1. **资源约束激发创造力**
> "当你没有预算时，你会被迫变得有创意。这往往导致更好的结果。"

- 不要抱怨资源不足
- 把它视为创意约束
- 一些最好的营销来自资源有限的公司

### 2. **专注于高杠杆活动**
- 不是所有营销活动价值相同
- 有些产生 10x 回报，有些几乎没有影响
- 学会识别和专注于高杠杆活动

### 3. **真实胜于完美**
- 真实的用户故事比完美制作的广告
- 真实的互动比精心策划的信息
- 人们可以感知真实

### 4. **速度和迭代**
- 不要等待完美的策略
- 开始、测量、调整
- 市场变化快，你需要更快

### 5. **用户至上，渠道次之**
- 从用户需求开始，不是渠道
- 如果用户成功，增长会跟随
- 不要本末倒置

### 6. **长期价值建设**
- 不要只看短期指标
- 品牌和声誉需要时间
- 今天做正确的事，未来会受益

## 实战手册

### 初创公司营销优先级矩阵

**按阶段划分**：

**种子期（Pre-PMF）**：
- [ ] 产品市场匹配验证
- [ ] 用户访谈和研究
- [ ] 与早期用户的直接沟通
- [ ] 基本网站和解释

**早期（Post-PMF，<1M ARR）**：
- [ ] 客户成功和留存
- [ ] 推荐计划
- [ ] 内容营销（教育性）
- [ ] 社区建设
- [ ] 精准付费广告测试

**成长期（1-10M ARR）**：
- [ ] 扩大有效的渠道
- [ ] 雇佣第一个营销人员
- [ ] 建立营销基础设施
- [ ] 企业功能（如适用）
- [ ] 品牌建设

**什么不要做**：
- ❌ 过早 PR 公司
- ❌ 在所有渠道上竞争
- ❌ 雇佣大公司营销人员
- ❌ 过度关注品牌而忽略增长

### "以小博大"策略检查清单

**创意营销**：
- [ ] 我们可以做什么是大公司不会做的？
- [ ] 我们有什么独特故事可以讲述？
- [ ] 我们如何创造意想不到的体验？

**社区利用**：
- [ ] 谁是我们最热情的用户？
- [ ] 如何给他们平台和工具？
- [ ] 我们如何奖励倡导者？

**差异化**：
- [ ] 我们的信息独特吗？
- [ ] 我们的视觉风格独特吗？
- [ ] 我们的用户体验难忘吗？

### 客户成功 = 营销实践

**每周同步**：
1. 营销：本周获取的承诺是什么？
2. 客户成功：这些承诺兑现了吗？
3. 共同：什么让客户最成功？
4. 共同：如何找到更多这样的客户？

**共享指标**：
- 不只是"获客数量"
- 而是"成功获客数量"
- 定义"成功"（激活、留存、扩张）

**反馈循环**：
- 客户问题 → 内容营销素材
- 成功案例 → 营销信息
- 营销承诺 → 客户成功预期管理

### VaccinateCA 风格的危机营销

**适用于初创公司**：

**清晰使命**：
- [ ] 我们的使命是什么？
- [ ] 它足够简单吗？
- [ ] 它足够紧迫吗？

**社区动员**：
- [ ] 谁会关心我们的使命？
- [ ] 我们如何让他们参与？
- [ ] 我们给他们什么工具？

**快速迭代**：
- [ ] 我们的最小可行版本是什么？
- [ ] 我们如何快速学习？
- [ ] 我们如何快速改进？

### 早期营销招聘

**面试问题**：
1. "描述一个你用很少预算产生重大影响的例子"
2. "你会如何衡量这个营销活动的成功？"
3. "如果产品有问题，你如何处理营销？"
4. "你如何决定什么营销活动优先？"

**寻找信号**：
- ✅ 理解指标和实验
- ✅ 可以自己执行（不只是管理）
- ✅ 产品好奇心
- ✅ 用户同理心
- ❌ 只关注品牌建设
- ❌ 依赖大型团队
- ❌ 不理解产品

### PR 和发布指南

**何时忽略 PR**：
- PMF 之前
- 有更高杠杆的活动
- 预算紧张
- 产品准备不足

**何时考虑 PR**：
- 有真正新闻（融资、里程碑）
- 有独特角度
- 可以借势
- 有资源跟进

**如何做 PR（如果要做）**：
- 直接与记者建立关系
- 准备好真实故事，不是推销
- 不要期望即时回报
- 将其视为品牌建设，不是主要增长引擎

### 避免类别创造陷阱

**检查清单**：
- [ ] 用户是否已经在寻找解决方案？
- [ ] 现有类别是否足够描述我们？
- [ ] 我们有足够预算教育市场吗？

**如果必须创造新类别**：
- 确保有独特且真实的不同
- 准备好花大量时间教育
- 用现有类别作为桥梁
- 专注于问题，不是词汇

**更好的方法**：
- "我们让 [常见任务] 变得 [更好/更快/更容易]"
- "像 [已知产品]，但 [关键差异]"
- 专注于价值，不是定义

## 反常识视角

### 1. **"品牌建设太早"的迷思**
**常见观点**：早期公司不应该关注品牌
**现实**：
- 品牌不只是 logo 和颜色
- 品牌是你如何对待用户、你如何沟通、你的价值观
- 从第一天开始就在建设品牌，不论你是否意识到
- **更好的思考**：从第一天起有意建设品牌，即使预算小

### 2. **"客户成功只是支持"**
**常见观点**：客户成功 = 回答问题
**现实**：
- 客户成功是营销和产品的耳目
- 他们最了解什么让用户成功和失败
- 他们应该是产品改进和营销策略的主要输入
- **更好的思考**：客户成功是战略功能，不只是支持团队

### 3. **"需要到处都有 presence"**
**常见观点**：应该在所有社交媒体平台活跃
**现实**：
- 资源有限，不能什么都做
- 最好精通一个渠道，而不是平庸地在多个渠道
- 选择你的用户所在的地方
- **更好的思考**：深度优于广度

### 4. **"高端周边太贵"**
**常见观点**：初创公司不应该在周边上花太多钱
**现实**：
- AirPods ($150) vs T 恤 ($10)
- AirPods 创造了 10x 的记忆和影响力
- 人们保留和使用 AirPods，T 恤在抽屉里
- **更好的思考**：质量胜于数量，影响力胜于覆盖

### 5. **"SEO 应该驱动内容"**
**常见观点**：写搜索量高的内容
**现实**：
- Airtable 的模板是为了用户成功，不是 SEO
- 如果用户成功，SEO 会跟随
- 如果你只为 SEO 写内容，用户不会成功
- **更好的思考**：用户价值第一，搜索量第二

### 6. **"B2B 必须从 B2B 开始"**
**常见观点**：B2B 产品需要立即关注企业销售
**现实**：
- Airtable 从 B2C 开始，B2B 自然跟随
- 让个人用户成为你的销售人员
- 当有需求时建设企业功能
- **更好的思考**：让产品价值驱动采用，不是销售策略

### 7. **"PR 是必需的"**
**常见观点**：每个公司都需要 PR
**现实**：
- 大多数成功的初创公司 PR 很少
- PR 很少驱动显著增长
- PR 花费时间，时间应该花在高杠杆活动上
- **更好的思考**：PR 是可选的，大多数公司应该忽略或最小化

### 8. **"需要完美的营销策略"**
**常见观点**：在执行之前需要完整策略
**现实**：
- 市场变化太快
- 最好的营销来自实验和迭代
- 完美策略的幻觉是执行的敌人
- **更好的思考**：有方向，然后开始实验和调整
