# Shaun Clowes - Product Leadership Excellence 播客摘要

## 嘉宾简介
Shaun Clowes是Confluent的首席产品官,曾任MuleSoft首席产品官(Salesforce价值10亿美元业务),Metromile(上市汽车保险公司)首席产品官,在Atlassian工作6年负责Jira敏捷开发并建立第一个B2B增长团队。他创建了两门最流行的Reforge课程:留存与参与度、产品经理数据课程。

## 核心议题
1. **PM职业现状** - 为什么15-20年后PM仍然是相对不发达的学科
2. **优秀PM特征** - 10倍PM的杠杆效应和可复制性
3. **AI对PM的影响** - 数据管理是最重要的影响领域
4. **客户洞察** - 80/20法则:80%时间思考建筑外的事情
5. **反馈之河(Feedback River)** - 用LLM构建的洞察系统

## 核心洞察(比喻/英文术语)

### PM学科的不发达状态(Undeveloped PM Discipline)
**"如果是医生,你会说这完全不可接受"**
- 15-20年进入PM,结果仍然是随机的
- 结果随机,行为随机,个人表现看似随机
- 没有触及真正重要的事情、真正增加价值的事情
- 抽象推理所需的东西不工作
- **10倍工程师 vs 10倍PM**: 如果有10倍工程师,必须有10倍PM
- **10倍PM的杠杆**: 10倍资源 × 10倍PM = 100倍回报或更多

### PM的工作是说90%不(PM's Job is Saying No 90% of the Time)
- 从一开始就让你成为坏人
- 必须快速获得runs on the board
- 必须证明有正确的洞察、正确的数据、做出正确的决定
- 否则没有另一次机会
- 这就是为什么PM最容易受到批评和 singled out

### 产品管理的乐趣(PM is About Fun)
**"试图找到国际象棋棋盘上未被占据但有价值的部分"**
- 在不确定下做决定
- 非常有趣但也非常痛苦和令人沮丧
- 很难说服人们

### 80/20法则:建筑外思考(80% Time Outside the Building)
- Steve Blank说应该花80%时间思考建筑外的事情
- 可能不在建筑外,但应该花80%时间思考建筑外
- **极少数PM这样做**: 他们被拖入内部政治、scrum管理、产品交付
- **你永远无法获得A**: 因为你没有从根本上解决工作
- **工作不是执行**: 而是找到可靠的、差异化的价值,你可以独特地交付到市场

### 客户视角第一(Always Start from Outside-in)
**每个文档,每件事,总是从客户视角、市场视角、竞争对手视角说话**
- 听从这个建议的PM几乎立即变得更好
- 因为他们从一个更容易理解的地方开始
- 其次是数据知情

### 避免可用性和确认偏差(Avoid Availability & Confirmation Bias)
- 大多数人去找他们总是交谈的人,学不到特别新的东西
- 不综合结果
- 不寻求counterfactual
- 不寻求证明自己错的证据
- 不分析竞争对手在做什么
- 不带回产品如何实际使用的数据 vs 人们说的如何使用
- **所有数据没有分析不是很有用**

### AI工具的真正价值:数据管理(AI's Impact = Data Management)
**"LLM是无限制的信息食用者"**
- 你永远无法给LLM足够的信息来真正获得其价值
- 给它越多,它变得越好
- 需要弄清楚如何带来尽可能多的关于客户、请求、竞争对手的信息
- 将所有这些带给LLM
- **模型 vs 数据**: Anthropic发现所有成功来自模型研究和提供给模型的数据,而不是优化UX

### 反馈之河(Feedback River)
**Sachin Rekhi的概念:聪明的PM不断在反馈河中游泳**
- 用用户访谈数据、直接客户反馈、NPS数据、竞争对手信息包围自己
- LLM和基于它的工具可以exceptionally good for this
- Confluent使用LLM处理大量入站客户请求:
  - 总结它们
  - 找到类似的请求(语义方式)
  - 跨数百或数千入站反馈看最流行的想法vs最不流行的

### LLM作为策略挑战者(LLM as Strategy Challenger)
**不要寻找你希望看到的,寻找你不寻找看到的**
- 把客户访谈放入ChatGPT
- "嘿ChatGPT,这是我的策略。告诉我我的策略在哪里不符合这些客户所说的。"
- 询问客户在探测你试图做什么的边缘
- 询问客户说什么更符合竞争对手的说法
- 复制粘贴竞争对手的定位文档
- "这比我的东西更适合他们所说的吗?"
- **通常很creepy**: "哦,他们可能会做这个,他们可能会做那个"

## 心法与原则

### 1. 寻找差距和机会(Find Gaps & Opportunities)
- 竞争优势从提取出其他人看不到的东西中提取出来
- 找出我们哪里错了
- 找出一个精心放置的赌注可能有巨大回报的地方
- 不是带回随机的周二听到的综合版本

### 2. 从建筑外开始(Start from Outside-in)
- 每个文档,每件事
- 从客户视角、市场视角、竞争对手视角
- 不要从内部政治、scrum执行、产品交付开始

### 3. 数据知情(Data Informed)
- 不要只是数据驱动
- 用数据inform决策
- 但从客户和市场视角开始

### 4. 挑战自己的假设(Challenge Your Own Assumptions)
- 寻求proof你错了
- 分析竞争对手
- 带回产品实际使用vs人们说的数据
- 综合结果

### 5. 构建反馈系统(Build Feedback Systems)
- 用LLM总结入站请求
- 语义匹配类似请求
- 识别最流行vs最不流行的想法
- 跨渠道追踪趋势

### 6. 让AI工具化Make AI Tooling
- 不要指望AI为你做工作
- 它们帮助你做工作中的复杂事情
- 找到差距、机会、共同线索
- 不一定都在你的大脑中完成

## 实战手册

### 成为优秀PM的路径

#### 第一步:转换视角
- 花80%时间思考建筑外的事情
- 从客户、市场、竞争对手视角开始
- 在每个文档中这样做

#### 第二步:避免偏差
- 不要只和你总是交谈的人交谈
- 寻求counterfactual
- 寻求证明你错的证据
- 分析竞争对手在做什么

#### 第三步:综合洞察
- 不要只是带回随机数据
- 提取其他人看不到的东西
- 找出你哪里错了
- 找出精心放置的赌注

#### 第四步:快速证明价值
- 说90%不,这样你可以说10%是
- 快速获得runs on the board
- 证明正确的洞察、数据、决策
- 否则没有另一次swing

### 使用AI/LLM提升PM工作

#### 客户访谈分析
```
1. 把访谈转录放入ChatGPT
2. "这是我的策略。告诉我我的策略在哪里不符合这些客户所说的。"
3. 询问客户在探测什么的边缘
4. 询问什么更符合竞争对手的说法
```

#### 竞争对手分析
```
1. 复制粘贴竞争对手公开定位文档
2. "总结他们的策略可能是什么"
3. "这比我的东西更适合客户所说的吗?"
4. 通常会给你crazy洞察
```

#### 入站反馈处理(Confluent方法)
```
1. LLM总结每个请求
2. 语义匹配类似请求(不是完全相同的词)
3. 跨所有入站需求识别趋势
4. 最流行想法vs最不流行
5. 什么变得更多流行vs什么变得更少流行
```

#### 来源
- 用户访谈数据
- 直接客户反馈
- NPS数据
- 竞争对手信息
- 产品使用数据

#### 方法
- 用这个信息包围自己

- 挑战你自己的假设

### B2B增长团队建设

#### 核心原则
- **效率驱动**: 用技术和数据驱动每个渠道
- **增长工程团队**: 支持销售效率
- **共享目标**: 工程师感到对管道驱动和渠道效率的所有权

#### 团队结构(Atlassian经验)
1. **渠道团队**: 付费营销、生命周期CRM、现场营销
2. **产品工程团队**: 支持增长和销售
3. **Skunk works团队**: 跨渠道创新实验

#### 指标
- 管道驱动
- 渠道效率
- 工程师对配额的所有权
- 中间和输入指标

- 10倍工程师存在
- 10倍PM必须存在

### 4. 寻找你不希望看到的
- 不要寻找你希望看到的
- 用ChatGPT找出你的策略在哪里不符合客户所说的
- 寻求证明你错的证据

### 5. 内部政治 = 无法获得A
