# Julian Shapiro - Lenny's Podcast 摘要

**嘉宾简介**: Julian Shapiro - Demand Curve创始人、前Webflow营销副总裁、全职投资人、作家，专注于增长策略和写作教育

## 1. 核心议题

- **Product-Led Acquisition（产品驱动获客）**：如何通过产品设计本身实现零成本获客和病毒式增长
- **State Building（状态构建）**：如何通过让用户在产品中积累不可转移的资产来提高留存
- **Novelty in Writing（写作中的新颖性）**：如何识别和创造让读者产生"哇"效应的内容
- **Topic Selection Framework（选题框架）**：如何选择值得写作的主题并确保完成度
- **Creativity Faucet（创造力水龙头）**：如何通过清空"坏想法"来获得优质创意

## 2. 核心洞察

### Product-Led Acquisition
- **"Don't start a startup where you need to go through someone else to get users"** - Paul Graham的这句话是Julian的核心投资哲学
- PLA vs. 病毒式传播：PLA是**产品使用本身带来新用户**，而非通过激励计划的推荐
- **零边际成本**：现有用户邀请新用户无需额外花费
- **避免第三方依赖**：不受Google算法更新或Facebook CPM波动的影响

### PLA的四大类别：
1. **Settling Debts（结算债务）**：如PayPal、Venmo - 接收方必须创建账户才能收到钱
2. **Conversation Lock-in（对话锁定）**：如WhatsApp、Slack - 必须加入应用才能参与对话
3. **Billboarding（广告牌效应）**：
   - 签名档（"Sent from my iPhone"）
   - 可见性产品（Tesla、AirPods、Nike鞋）
   - 功能性分享（Calendly链接必须分享才能使用）
4. **User Generated Content（用户生成内容）**：如YouTube、TikTok、Quora - 用户创建内容时为平台做广告

### State Building（状态构建）
- **"The rich get richer"（富人越富）** - 用户在产品中投入越多，离开成本越高
- **Non-transferable Reputation（不可转移声誉）**：eBay卖家的10,000条好评无法带到竞争对手平台
- **Non-transferable Audience（不可转移受众）**：YouTuber的订阅者无法导出
- **Social Graph Investment（社交图谱投资）**：在Facebook花费10年添加的朋友无法带走
- **Embedded Infrastructure（嵌入式基础设施）**：Twilio、Stripe、AWS - 切换需要重写代码

### Novelty Framework（新颖性框架）
**写作质量 = Novelty × Resonance**

**Novelty的五大类型**：
1. **Counterintuitive（反直觉）**："世界并非由杰出人士运行"
2. **Counter-narrative（反叙事）**："读很多书是成年人最社会接受的虚荣指标"
3. **Shock and Awe（震撼）**：令人震惊的事实
4. **Elegant Articulation（优雅表达）**：Naval式将复杂思想浓缩为金句
5. **Resonance（共鸣）**：通过例子、类比、故事让新颖观点深入人心

## 3. 心法与原则

- **"Bad ideas is progress"（坏想法就是进步）** - 创作过程中必须先清空所有坏想法，好想法才会涌现
- **质量 > 数量**：宁愿要10,000个mind followers（追随你思想的人）而非100,000个labor followers（追随你劳动的人）
- **Evergreen Content（常青内容）**：只写会长期相关的内容，避免追逐热点
- **Force Function（强制函数）**：Handbooks是强迫自己彻底学习某主题的方式
- **Objective + Motivation（目标+动力）**：完成写作的方程式

## 4. 实战手册

### 如何构建PLA产品
1. **在产品路线图阶段就考虑PLA**：选择天生具备PLA特性的创意
2. **评估现有产品**：问自己"我的产品是否支持这四类PLA中的任何一种？"
3. **添加结算债务功能**：如果适用，让用户必须邀请他人才能完成交易
4. **植入社交功能**：考虑添加聊天或协作功能
5. **利用自己的表面面积**：在用户生成的内容中添加品牌水印

### 如何提高留存（State Building）
1. **鼓励声誉积累**：评分、评论、徽章系统
2. **构建受众图谱**：让用户建立follower关系
3. **社交图谱锁定**：好友网络、连接
4. **数据/内容沉淀**：用户的历史数据、创作内容
5. **集成深度**：成为工作流程或技术栈的一部分

### 如何写出高Novelty内容
1. **收集Novelty瞬间**：每次读到让你"哇"的内容，记录并打分（0-5）
2. **20朋友测试法**：让20个朋友阅读草稿，高亮让他们产生"哇"效应的句子
3. **可视化Novelty密度**：查看哪些段落缺乏Novelty，压缩或删除
4. **识别Novelty类型**：检查属于反直觉、反叙事、震撼还是优雅表达
5. **增加Resonance**：加入具体例子、类比、故事

### 如何选择写作主题
**目标 + 动力 = 完成**

**Objective（目标）示例**：
- 揭穿现状，打开人们眼界
- 表达大家都在想但没人说的事
- 分享原创研究和实验洞察
- 讲一个有启发性的情感故事

**Motivation（动力）示例**：
- 一吐为快（get off my chest）
- 解决困扰自己的难题
- 对某个主题的痴迷

### Creativity Faucet（创造力水龙头）实践
1. **接受坏想法**：每个创作 session 开始时，写下所有坏想法
2. **不要评判**：坏想法是必经阶段，不是失败
3. **持续输出**：直到坏想法清空，好想法自然涌现
4. **迭代优化**：从弱模仿开始 → 识别弱点 → 迭代到原创
5. **时间投入**：预留足够时间（可能需要1小时）清空坏想法

## 5. 反常识视角

- **Threads虽有效但不推荐**：Julian认为fortune cookie threads（鸡汤推文串）会吸引labor followers而非mind followers，降低受众质量
- **推荐计划≠PLA**：推荐计划是人工激励，PLA是产品使用自然带来增长
- **"Moat"（护城河）被误用**：真正的moat是政府关系或科学突破，大多数公司所谓的"moat"只是稍好的留存机制
- **读大量书≠聪明**："Reading many books is the most socially accepted vanity metric for adults" - 真正重要的是高效学习和创造
- **世界不由杰出者运行**：Imposter syndrome的真正原因是拒绝接受"你的世界级同行并不特别，只是更有纪律"

- 所有内容视为**evergreen（常青）**
- 将handbooks作为自己的参考手册
- 强迫症式的简洁性追求
- 批评：写作过于dry，缺乏resonance（故意为之）

- **Book**: *On Writing Well* - 教会他删除2/3的内容以直达核心
