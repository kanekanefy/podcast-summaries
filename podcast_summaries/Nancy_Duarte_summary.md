# Nancy Duarte - Lenny's Podcast 摘要

## 嘉宾简介

Nancy Duarte是畅销书作者、演讲者和Duarte Inc.的CEO。她的公司帮助创造了超过250,000场演示文稿，客户包括Apple、TED、Google、世界银行，最著名的是为Al Gore制作《难以忽视的真相》演示文稿。她被誉为沟通和演讲领域的权威专家。

## 核心议题

1. **演示文稿的核心原则** - 听众是英雄，同理心至上
2. **故事结构的力量** - "是什么"vs"可能是什么"的对比结构
3. **视觉化沟通** - 让人们看到你在说什么
4. **克服演讲恐惧** - 从紧张到自信的转变技巧
5. **变革领导力** - 火炬手领导者的五阶段结构

## 核心洞察

### 1. 听众是英雄 - 同理心的力量
**Nancy的童年影响**:
```
母亲是临床自恋症患者
→ 缺乏同理心基因
→ 这种缺失让Nancy不断追求同理心的重要性
→ 所有书籍和模型都以同理心为核心
```

**传统误区 vs 真相**:
```
误区: 演讲者是英雄
- 演讲者站在舞台中央
- 灯光聚焦在他们身上
- 他们说得最多

真相: 听众才是英雄
- 听众有权接受或拒绝你的想法
- 权力平衡在听众手中，而非演讲者
- 演讲者应该是"导师"(Mentor)角色
```

**神话和电影中的导师原型**:
- Obi-Wan Kenobi给Luke Skywalker两样东西：
  1. 光剑（外在旅程的工具）
  2. 决心（通过原力获得的内在工具）

**演讲者的角色**:
```
听众有内在冲突
→ 你需要给他们东西来安抚
→ 然后要求他们采取行动

如果还没有同理心地考虑
你要求他们做的事情有多困难
→ 他们就不会为你做这件事
```

### 2. "南瓜牙齿"故事结构 - TED演讲成名作
**Nancy的三年研究之旅**:
```
方法:
→ 不是数字原住民
→ 使用1/4英寸方格纸
→ 手动分析经典演讲
→ 写下每一个字
→ 需要以模拟方式"看到"它

发现:
→ 最伟大的演讲都有起伏
→ 不是单一故事
→ 但有"是什么"vs"可能是什么"的节奏
```

**视觉化结构**:
```
屏幕底部画一条线（从左到右）
→ 这条线代表"是什么"(What is)
→ 然后直线向上移动到"可能是什么"(What could be)
→ 回到底部线，再说"是什么"
→ 向上"可能是什么"
→ 重复这个模式
→ 最后的"可能是什么"成为"新的极乐"(New bliss)

创造渴望:
→ 这种在"是什么"和"可能是什么"之间穿梭
→ 通过对比创造渴望
→ 让人们离开当前状态/现状
→ 让他们渴望这个未来状态
```

**经典例子 - Steve Jobs的iPhone发布会**:
```
"是什么": 公司现状
→ "我们的店比10个Macworld博览会的还要满"

"是什么"vs"可能是什么"的快速对比:
→ 看看现在Blackberry有多糟糕
→ 一旦你看到了我们在做什么
```

**历史验证**:
- Nancy分析了Martin Luther King Jr.的演讲
- 分析了Steve Jobs的iPhone发布会演讲
- 发现相同的对比节奏和脉动
- 当她在两者中找到相同模式时，知道自己解开了故事之谜

### 3. 日常生活中的对比框架
**"肮脏的小秘密"**:
> "很多人认为只有在大舞台演讲时才需要好好演示。我要告诉你一个肮脏的小秘密：我可以用一个快速的'是什么-可能是什么-新的极乐'让我老公周末做家务。"

**任何影响时刻都有效**:
- 会议中
- 电话中
- 任何有影响力在空气中的时刻
- 如果你练习得足够多，它会作为心理模型存在于你的脑海中

**Nancy和老公的沟通案例**:
```
早期婚姻问题:
→ 纠纷通常只关于流程
→ 我执行某事的方式与他不同
→ 例如："你为什么要那样切洋葱？"

解决方案:
→ "你想切洋葱还是让我按我的方式切？"

"是什么-可能是什么-新的极乐"的实际应用:

Nancy需要老公帮忙:
"是什么"（需要更长，因为他注重细节）:
→ "天哪，明天我有连续的会议
   就在这个时候，我还要上Lenny的播客
   那是狗狗最粘人的时候
   如果这不发生，我就要改期到下周
   下周已经排满了
   你知道我在一天结束时压力大
   是多么难相处"

"可能是什么":
→ "狗狗 daycare，她上次很喜欢
   她和一只红色的骑士查理王小猎犬一起睡觉
   她很喜欢"

"新的极乐":
→ （可能是任何婚姻承诺）
```

**爱的语言在公司中的应用**:
- Duarte公司的每个人都知道彼此的爱的语言
- 这个人通过书面笔记感觉更被欣赏
- 那个人通过礼物感觉更被欣赏
- 每个人都知道
- 这融入了他们的婚姻、公司、运作方式

### 4. 视觉化沟通的力量
**核心原则**: "他们能看到我在说什么吗？"

**Star Moment (Something They'll Always Remember)**:
- 戏剧性的数据展示
- 引人入胜的故事
- 美丽的图片
- 通过图片实现的对齐

**实际应用**:
```
技术公司的例子:
→ 用图片描述你的产品
→ 这个东西在里面、外面、附在上面、在它上面、在它上面吗
→ 尤其是架构幻灯片或技术如何在复杂系统中流动

当人们能看到它
→ 伴随你的口头叙述
→ 他们能实际理解你传达的内容
→ 继续前进

如果只有口头叙述:
→ 效果不会那么好
```

**餐桌/白板技巧**:
- 在餐桌上有趣的对话中
- 想让别人看到你在说什么
→ 拿出餐巾纸画下来
- 会议中有人走到白板前画某事
- Nancy的设计团队特别擅长这个：
  - 站起来说"我想画给你看我的想法"
  - 因为你刚才口头说了
  - 我看到了这个
  - 这是你的意图吗？
- 整个房间站起来开始共同创作图形
- 每个人都看到完全相同的东西、步骤、洞察和顺序
- 没有人带着疑问离开

**领导者视觉化挑战**:
```
问题:
→ Nancy看到完整的美丽画面
→ 但只提供了一小部分马赛克砖
→ 团队看到22块马赛克砖

解决方案:
→ 真正把它落实到地面
→ "这是达到这个惊人结果的7个步骤"
→ 有时我们在心目中看得太清楚

案例:
→ 与一位著名强大的CEO合作
→ 她说话时，手臂以特定方式移动
→ "我可以看出你心目中有一个画面"
→ 走上前，画出这个、这个、这个
→ "正是！"
→ 她被带进来是因为没人能表达她心目中看到的东西
→ 这是向10万零售员工推出的庞大项目
→ 一旦人们能看到她在说什么
→ 突破就开始发生
```

### 6. 幻灯片设计战术建议

**每张幻灯片一个观点**:
```
整体结构:
→ 整个演示文稿应该基于"听众旅程"
→ 大想法(Big Idea): 你的观点是什么
→ 如果他们采用或不采用会有什么风险？
→ 这是整个幻灯片组的组织机制

每张幻灯片:
→ 支持那个大想法的每个幻灯片本身应该提出一个观点
→ 人们一次不能处理太多事情
```

**Slide Doc (幻灯片文档)概念**:
```
场景: 你可以制作一个演示文稿
      只是通过电子邮件分发
      每个人都能理解

怎么做:
→ 放更多文字
→ 更强的图片
→ 可以有100页附录
→ 前面可能只有5张幻灯片
→ 但他们看到你思考的所有东西都跟在后面

比较:
→ 类似Amazon流行的6页备忘录
→ Nancy认为幻灯片中的文字和图片更好
```

**知道你的受众**:
```
某些受众想要高密度幻灯片:
→ 因为那是他们文化中沟通的方式

如果你出现舞台就绪的幻灯片:
→ 他们会把你笑出房间

你必须:
→ 了解你的受众
→ 了解他们如何沟通
→ 他们和谁说话
→ 映射到那个
```

### 7. 演讲恐惧症 - 克服紧张

**为什么我们会害怕**:
```
战斗或逃跑本能:
→ 走上舞台
→ 你的身体、思想、灵魂感到受威胁
→ 就像被动物攻击一样
```

**Nancy的观察**:
```
紧张的人通常有更好的内容:
→ 他们通常真的深入思考
→ 像Lenny一样是模式寻找者
→ 喜欢做有思想的工作

那些"我got this"的人:
→ 可能只是即兴发挥
→ 走上舞台
```

**实用技巧**:

**1. 视角转换**:
```
坐观众席的某个座位:
→ 看舞台
→ 看场景
→ 想象自己在上面
→ 把自己想象成那个友好的面孔
→ 那个很高兴见到你的人
→ 那个对你说话感到高兴的人

当你站起来时:
→ 记住你看到自己坐在那里微笑
→ 非常高兴
→ 改变你的视觉模型
→ 人们的脸不会皱眉
→ 不会评判你
→ 不会怀疑你
→ 这些都只在你脑子里
```

**2. 个人化准备仪式**:

**Nancy的调查结果**:
- 一些公共演讲者："我听重金属音乐并在整个会议中心跳来跳去，让自己兴奋起来"

**Nancy的方法（冷静型）**:
```
找一个黑暗角落:
→ 不去化妆间
→ 不喜欢听到闲聊
→ 必须专注于内容
→ 找到后台最黑暗的角落
→ 平静地坐着
→ 只是呼吸

呼吸模式:
→ 深吸一口气
→ 肺满时再吸一口气
→ 非常缓慢地呼出

化学转移技巧:
→ 看有趣的视频清单
→ 从不观看（保持新鲜感）
→ 上舞台前看
→ 化学上，整个身体从紧张转变为笑声
→ 真的有帮助
```

**尝试错误的经历**:
- Nancy尝试了"重金属和跳跃"方法
- 跳跃，伸展，手臂伸向空中
- 走上舞台
- 助理接到电话："我们担心Nancy，她可能需要看医生，她永远无法控制她的呼吸"
- 只是因为她稍微给自己打了一点气
- 回到平静、沉思、冥想的演讲前仪式

### 8. 火炬手领导力 - 五阶段变革结构

**为什么是火炬**:
```
在洞穴中拿着火炬:
→ 你只能看到大约5-8英尺周围
→ 但这足以驱散跟随你的人的恐惧

没有人清楚地看到未来:
→ 没有人有那种级别的技能
→ 我们只知道需要朝这个方向穿越
→ 在未来处于正确的位置
→ 所有员工都安全
→ 我们保持行业领导地位
```

**五阶段结构 (Dream, Leap, Fight, Climb, Arrive)**:

**1. Dream (梦想阶段)**:
```
行动:
→ "嘿大家，这是我们前往的未来之地"
→ 这是Nancy在启动会议上所做的

你需要:
→ 口头表达梦想
```

**2. Leap (跳跃阶段)**:
```
行动:
→ 他们选择跳入并跟随你
→ 或者选择不跟随

例子:
→ Frodo和Sam
→ 只有几个霍比特人跟随他
→ 人们选择承诺这段旅程
```

**3. Fight & Climb (战斗与攀登阶段 - 中间混乱)**:
```
现实:
→ 他们承诺你的想法、项目
→ 起初很兴奋
→ 然后进入"天哪，这比我想象的更难"状态
→ 长期艰苦的跋涉
→ 攀登让人筋疲力尽
→ "我不知道我是否有这么多战斗能力"

不是与彼此战斗:
→ 而是克服这个路障，那个路障
→ 必须去争取那个预算

模式:
→ 战斗、攀登、战斗、攀登、战斗、攀登
```

**4. Arrive (到达阶段)**:
```
最终:
→ 你到达了
```

**每个阶段需要**:
```
使用:
→ 演讲
→ 故事
→ 仪式
→ 符号

目的:
→ 给跟随你的人情感燃料
→ 让他们继续前进
→ 看到这个想法实现
```

**为什么现在特别重要**:
- 引领变革一直在发生
- 尤其是过去几年
- 变革、变革、变革
- 人们真的很依赖这本书的内容

问题:

## 心法与原则

### 三大核心原则

**1. 让听众成为英雄**
- 从同理心开始
- 问自己：我在和谁说话？
- 我将如何帮助他们摆脱困境？
- 带来魔法工具？

**2. 用故事注入演讲**
- 利用故事结构（开头、中间、结尾）
- 融入"起伏"故事结构
- 建立张力并释放它
- 故事创造渴望
- 让人们渴望以前从未想要过的东西

**3. 问"他们能看到我在说什么吗？"**

### Duarte公司的工作方式
```
像皮克斯制作电影一样制作演示文稿:

流程:
→ 创作叙述
→ 创作大想法
→ 创作脚本
→ 可视化某些时刻
→ 开始映射
→ 开始分解

大型模型:
→ 有时关键模型首先在演示文稿中作为想法发生
→ 立即开始处理一些关键模型
→ 开始在公司周围循环
→ 每个人都必须建立共识

多重运动:
→ 草图这个
→ 离开，和那个部门工作
→ 尝试解决这个问题
→ 解决这个
→ 在最后重新组装
→ 叙述是你解决所有问题的地方
→ 当他们站立和交付时
→ 就是那个所有过程支持的声音轨道
```

## 实战手册

### 内部演示文稿的准备过程
**Nancy自己的内部演示**:
```
最难的所有演示:
→ 需要目标或达成目标的年度愿景谈话

准备:
1. 听察之旅 (Listening Tour):
   → 基于调查
   → 基于访谈
   → 将这些信息反馈
   → 与我们要他们做的事情进行比较
   → 做差距分析

2. 粗糙剪辑:
   → 执行团队创建真正的粗糙剪辑
   → 邀请下一层领导者
   → 假演示（幻灯片很丑）
   → 不在幻灯片上花时间
   → 这是关于信息
   → 可能有一两个或三个模型

3. 反馈:
   → 很难从粗糙剪辑
   → "这是我们要说的"
   → 到让它真正共鸣

4. 交付:
   → 在所有工作完成后
   → 与公司分享
```

**团队反应差异**:
```
旅行和公开演讲:
→ "天哪，我爱你的模型"
→ "天哪，我可以和你合影吗？"

站在自己团队面前:
→ "我想知道她要说什么"
→ "她要么会让我的工作更难"
→ "要么她会改变我的优先事项"
→ 他们带着更多怀疑进入
→ 绝对掌握了年度启动会议
→ 然后做季度更新
→ 一种节奏
→ 人们变得兴奋
→ "我们现在真的在杀死它"
```

### 制作幻灯片的具体建议

**如果你要创建幻灯片**:
```
1. 首先思考（不要立即打开软件）:
   → 改变你的环境
   → 许多人会启动演示文稿（非常线性）
   → 一张幻灯片、第二张幻灯片、第三张幻灯片

2. 思考和规划:
   → 画故事板
   → 第一点、第二点、第三点
   → 只是先思考
   → 可以是模拟或数字
   → 在所有演示文稿前放一页
   → 只是方框
   → 只是让叙述正确

3. 打开软件时:
   → 什么幻灯片类型最能传达这一点？
   → 是表格？放表格
   → 特别是项目经理
   → 必须传达密集的项目信息

与团队对齐:
   → 如果你和同伴在一个房间里
   → 每个人都有自己的简写和工作方式
   → 把那个公共幻灯片放上去
   → 团队的常见幻灯片可能对外面的世界很密集
   → 但每个人都习惯使用它
   → 使用常见的、可接受的框架、幻灯片或表格或Excel电子表格没有害处
   → 你在对齐一个流程
   → 不要觉得每一张都需要小猫的电影般图片
   → 那不会让你到任何地方
   → 你试图移动一个目标
   → 这确实意味着你的幻灯片可能更密集
   → 有时内部幻灯片有更多重要信息需要推产品或推流程
```

## 反常识视角

### 1. 生产质量 vs 内容价值
**传统认知**: 高制作价值 = 更好的影响力
**Nancy的发现**: 内容实在的、未制作的内容比高制作价值内容表现更好

### 2. 演讲恐惧的价值
**反直觉发现**: 紧张的演讲者通常有更好的内容
- 他们真的深入思考
- 像Lenny这样的模式寻找者
- 喜欢做有思想的工作
- "我got this"的人可能只是即兴发挥

### 3. 幻灯片的误区
**不是每次都需要幻灯片**:
- 1亿美元预算案例：不要让技术挡在你和CEO之间
- 有时白板或纸张更有效
- 让他们看到你的眼睛、你的激情

### 4. 正式vs非正式
**Nancy的个人风格**:
- 不去化妆间（green room）
- 不喜欢听到闲聊
- 必须专注于内容
- 找到后台最黑暗的角落
- 平静地坐着，只是呼吸

**相反的风格**:
- 听重金属音乐
- 在整个会议中心跳来跳去
- 让自己兴奋起来
- 对某些人有效

### 5. 婚姻沟通与商业演示的联系
**肮脏的小秘密**: 同样的对比框架("是什么-可能是什么-新的极乐")适用于：
- 大舞台演讲
- 会议
- 电话
- 让老公做家务
- 任何有影响力在空气中的时刻

### 6. 过度设计的失败
**Mac进入市场的时机**:
- 图形设计师拒绝使用（势利）
- "这是玩具，很丑，是位图的"
- Duarte在设计师拒绝时进入
- 借图书馆关于排版的书籍
- 弄清楚能用这个工具做什么
- 当工具进入大众手中时，它走向了错误的方向
- 需要重新塑造媒介
