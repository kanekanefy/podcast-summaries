# Naomi Gleit 播客总结

## 嘉宾简介

Naomi Gleit是Meta的产品负责人，除Mark Zuckerberg外任职时间最长的执行官，作为第29号员工已工作近20年，见证了公司从30人到1.5万亿美元市值的整个历程。

## 核心议题

1. 如何作为非技术背景的员工进入并成长为顶级产品经理
2. 从Facebook早期增长团队学到的核心经验
3. 作为Meta "Conductor"的独特管理哲学和"Naomi-isms"框架
4. 与Zuckerberg共事20年学到的重要领导力经验
5. Teen Accounts项目的复杂跨职能协作案例

## 核心洞察

**比喻/英文术语：**
- **PM as Conductor (指挥家)** - PM是指挥家，不是主角，而是让所有职能正确演奏
- **Extreme Clarity (极致清晰)** - 所有人理解相同的事实，即使不同意
- **Canonical Everything (标准化一切)** - 唯一标准文档、术语、会议、聊天群
- **Disagreeable Givers** - 最珍贵的组织成员，愿意反对但真心为公司好
- **Naomi-isms** - Naomi反复使用的工作框架集合
- **Growth Accounting (增长核算)** - 新增 - 停滞 + 复活 = 净增长
- **Seven Friends in Ten Days (7天10好友)** - 激活指标
- **School Pyramid (学校金字塔)** - 从幼儿园到博士的简化框架

**关键洞察：**

1. **"火箭船理论"的实践者** - Cheryl Sandberg说"当火箭船时不要问什么座位"，Naomi通过5-10次cold walk-in最终进入Facebook
2. **增长团队的根本转变** - 从营销驱动增长转变为产品和数据驱动增长，这是行业首创
3. **PM是指挥家而非CEO** - "指挥家在演出中不说话"，PM的价值是协调而非创造
4. **标准化一切的哲学** - 唯一标准文档、唯一术语、单一负责人、标准会议流程
5. **80%问题是人和流程问题** - 只有20%是真正的战略或执行问题
6. **完美执行的重要性** - 只有完美执行才能评估战略是否正确
7. **Zuckerberg的四个生活课程** - 1.爱自己 2.才能服务他人 3.专注可控之事 4.永不放弃

## 心法与原则

### Naomi-isms框架

1. **Understand, Identify, Execute (理解-识别-执行)**
   - 2009年整个Q1只做数据instrumentation
   - "I guess when you can know" - 没有数据就无法做决策

2. **PM as Conductor**
   - 确保每个部分正确演奏
   - 保持和谐统一和正确节奏
   - PM不是明星，指挥家在演出中不说话

3. **Extreme Clarity (极致清晰)**
   - 所有人理解相同的事实
   - 可以不同意但基于相同数据
   - 避免因误解而产生的冲突

4. **Canonical Everything**
   - Canonical Doc - 项目信息的唯一来源
   - Canonical Nomenclature - 统一术语定义
   - Canonical Meetings - 标准会议流程
   - Canonical Chat - 唯一沟通渠道
   - Single-Threaded Owner (STO) - 每个工作流的单一负责人

5. **Simplifier Philosophy (简化者哲学)**
   - 从博士水平问题简化到幼儿园水平
   - 创建"学校金字塔"课程体系
   - 不是过度简化而是揭示基本构建块

### 领导力原则

1. **Learn-it-all, not Know-it-all**
   - Zuck是最快提升技能的人
   - 一年达到中文8年级水平
   - 从吉他到MMA到演讲的持续进化

2. **Disagreeable Givers > Agreeable Takers**
   - 最危险的人是看似nice但self-interested
   - 最珍贵的人愿意为公司利益反对

3. **Small Group文化**
   - 基于使命而非职业晋升
   - 每周一次战略会议(开放讨论)和一次运营会议(高度结构化)
   - 直接和诚实反馈的文化

## 实战手册

### 成为PM的路径

1. **Cold Walk-in策略**
   - 5-10次直接去办公室询问工作机会
   - 被Sean Parker的PA职位拒绝后继续尝试
   - 最终获得市场职位

2. **从非PM到PM的转换**
   - 自愿下班后"潜入"二楼(PM和工程师层)
   - 主动参与项目数月
   - 正式申请时已在做PM工作
   - 获得职位时二楼全体起立鼓掌(包括Boz)

### 运行高效会议

1. **会议前24小时**
   - 发送议程和预读材料
   - 包含三个选项和一个推荐
   - 使用Traffic Light框架(红绿灯评估)

2. **会议中**
   - 使用视觉材料实时编辑
   - 明确标记决定和下一步
   - 不使用bullet points，只使用numbered lists

3. **会议后24小时内**
   - 回复all发送会议记录
   - 使用日历邀请作为canonical沟通单元

### Traffic Light决策框架

- 三个选项作为行
- 评估标准作为列(legal, policy, privacy等)
- 红绿灯颜色编码
- 推荐选项应该有更多绿色/黄色
- 每个单元格包含具体rationale

### 处理复杂项目

1. **创建Canonical Doc包含**
   - Work Streams (工作流)
   - Owners (每个工作流的单一负责人)
   - Process (团队工作流程)
   - Canonical Meetings
   - Canonical Email List
   - Canonical Workplace Chat
   - Canonical Nomenclature
   - Canonical Visuals

2. **School Pyramid方法**
   - 建立幼儿园课程(基本构建块)
   - 建立小学课程
   - 建立中学课程
   - 建立大学课程
   - 最终达到博士水平运营

### 个人生产力系统

**四个Must-Haves:**
1. **Eat** - 营养摄入
2. **Sleep** - 8小时睡眠,眼罩,遮光窗帘
3. **Exercise** - 每天7点am非协商锻炼
4. **Alone Time** - 作为内向者的充电时间

**五个引体向上目标:**
- 设定清晰可衡量的目标
- "I can do hard things"的心理建设
- 技巧可能比力量更重要
- 信心转移到生活的其他方面

## 反常识视角

1. **"激活指标"的具体数字不重要**
   - 7天10好友或14天10好友都可以
   - 价值在于有一个清晰的共同目标
   - 不是具体数字而是retention goal

2. **增长团队可能被过度credit了**
   - PMF是最重要的因素
   - 增长团队很大程度上是在已有需求的基础上移除barrier
   - "We were relatively lucky to have high demand"

3. **PM应该更像是CEO而非不是CEO**
   - PM是最接近founder视角的角色
   - 应该channel CEO/founder的意图
   - "Great Non-Technical"时代PM必须证明价值

4. **不用bullet points而用numbered lists**
   - "You can never refer to a bullet"
   - "the second bullet, two up from the second"
   - numbered lists让会议中引用更清晰

5. **工作与生活的整合哲学**
   - "I'm training for life, not for an event"
   - 工作本身也是一种workout
   - 需要能move的身体状态

6. **Girls Just Want to Have Fun的科学**
   - 8小时睡眠可能比早工作一小时更productive
   - Four metrics of performance: eat, sleep, exercise, alone time
   - Wall Street Journal写了她的exercise regimen

**总结**: Naomi Gleit从Facebook第29号员工成长为Meta产品负责人的近20年历程，展现了非技术背景如何通过persistence和创造运气成为顶级PM。她的"Naomi-isms"框架(extreme clarity, canonical everything, PM as conductor)和对简化复杂问题的独特方法论，为PM和产品领导者提供了极其practical和可执行的tools。她强调了perfect execution、people和process的重要性，以及通过standardization和clarity来drive velocity和impact。同时，她也分享了personal performance系统(exercise, sleep, nutrition, alone time)如何支持high-pressure的career。
