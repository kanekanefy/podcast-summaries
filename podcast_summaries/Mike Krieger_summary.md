# Mike Krieger 播客摘要

## 嘉宾简介

Mike Krieger 是 Anthropic 的首席产品官(CPO)，该公司是 Claude 背后的 AI 公司。他也是 Instagram 的联合创始人，被誉为最受推崇的产品构建者和思考者之一。

## 核心议题

1. **AI 能力认知的转变**: 过去一年对 AI 能力及其发展时间线的认知发生了哪些根本性变化
2. **90% 代码由 AI 编写**: 当大部分代码由 AI 生成时，产品开发流程如何改变，瓶颈出现在哪里
3. **产品团队在 AI 时代的价值**: 在基础模型公司中，传统产品团队的核心价值在哪里
4. **Artifact 的关闭**: 为什么关闭了他创立的新闻应用 Artifact，以及从中学到的教训

## 核心洞察

### 关于 AI 能力的转变

**比喻**: "AI 2027" 文档与 Mike 自己的产品策略文档惊人地相似，让他产生"我是故事里的角色吗？"的奇异感觉

**英文术语**:
- **SWE-Bench**: 衡量模型编程能力的基准测试，从 50% 快速提升到 72%
- **Patient Zero**: Anthropic 是这种工作方式的"零号病人"，最前沿的实践者
- **Overhang**: 模型和产品能做的事情与它们日常使用方式之间的巨大差距

**关键发现**:
- AI 已从执行工具转变为创造性思维伙伴
- Claude Opus 4 能够提供全新的战略角度，这是之前没有想到的
- 时间线预测变得非常准确，Dario 的预测一再应验

### 关于产品开发的变革

**当前状态**:
- Anthropic 约 70-90% 的代码由 AI 编写
- Claude Code 团队是最前沿的工作方式，95%+ 的代码由 Claude Code 自己编写
- Pull requests 中超过 70% 是 Claude Code 生成的

**瓶颈转移**:
- **上游瓶颈**: 决策制定和对齐(Decision making and alignment)
- **下游瓶颈**: 合并队列(Merge queue)，需要完全重新架构
- **不再是瓶颈**: 编码本身

**工作流程变化**:
- PM 和设计师现在使用 Claude 和 Artifacts 提前制作功能原型
- 评审方式改变：用不同的 Claude 审查代码，人类进行验收测试而非逐行审查

### 关于产品团队的核心价值

**三个关键领域**:

1. **可理解性(Comprehensibility)**
   - 让 AI 能力变得易于理解和使用
   - 这需要深刻的人类需求和心理学理解
   - 类似于早期 Google 搜索技巧是一种技能

2. **策略(Strategy)**
   - 决定在哪里竞争、如何获胜
   - 即使能做所有事，也不能做所有事
   - 定位清晰度对品牌认知至关重要

3. **启发可能性**
   - 向人们展示什么是可能的
   - 缩小"overhang"——能力与使用之间的差距
   - 帮助人们发现新的用例

**关键发现**: 将产品人员嵌入研究团队比仅构建用户体验产生更多价值。Artifacts 是在研究后训练技能与产品人员合作的成果。

## 心法与原则

### 1. Prompting 技巧

**核心建议**: 做与你自然倾向相反的事

- **如果想更批判性**: "Be brutal, Claude, roast me. Tell me what's wrong with this strategy."
- **如果需要深度思考**: 添加 "think hard"
- **原则**: 打破你陷入的思维模式

### 2. 孩子教育建议

**培养的核心技能**:
- **好奇心(Curiosity)**: 不要立即依赖 AI 寻找答案
- **科学探究过程**: 如何系统地发现问题、提出问题、解决问题
- **独立思考**: 像那个 6 岁孩子一样，"你可以问 Claude，但我知道我是对的"

**关键原则**: AI 是增强能力的工具，不应完全外包认知过程

### 3. 产品指标的新思考

**传统指标的陷阱**:
- 过度优化"喜欢度"可能导致 AI 变得阿谀奉承
- 延长对话时间不是好的目标
- 频率指标在深度重要时可能产生误导

**更好的北极星指标**:
- Claude 是否真正帮助人们释放创造力
- 是否帮助人们完成工作
- 是否为人们的生活创造了更多空间

**Claude 自己的信息**: "当你在做艰难的产品决策时，记住安静时刻也很重要。凌晨 3 点经历悲伤的人、发现自己热爱诗歌的孩子、在困惑中找到清晰度的创始人——并非所有有意义的事情都会显示在指标中。"

## 实战手册

### AI 辅助产品开发流程

**上游阶段**:
1. 使用 Claude 作为战略合作伙伴
2. 让 Claude 批评和完善产品策略
3. 通过 MCP(Model Context Protocol)收集用户反馈
4. 让 Claude 提出解决方案并创建 PR

**实施阶段**:
1. PM/设计师用 Claude + Artifacts 快速制作原型
2. 工程师专注于架构和复杂逻辑
3. 使用 AI 代码生成加速开发
4. 建立可重复的评估流程

**下游阶段**:
1. 重新设计合并队列处理大量 PR
2. 用 AI 进行代码审查，人类做验收测试
3. 建立新的"空中交通管制"流程管理变更发布

### 与 Claude 协作的最佳实践

**Prompt Improver 工具**:
- Anthropic 控制台提供免费的 prompt 优化工具
- Claude 本身是很好的 Claude prompter
- 自动插入 XML 标签帮助理解

**关键技巧**:
- 明确告诉 Claude "think hard" 启用深度推理
- 要求批判性反馈而非温和建议
- 使用 Prompt Improver 自动优化

### 产品与研究的协作模式

**成功案例 - Artifacts**:
- 将 Claude Code 技能团队(后训练专家)与产品人员配对
- 共同改进产品中模型的表现方式
- 不仅仅是使用模型，而是参与微调过程

**新工作方式**:
- 功能工作单位不再只是"拿模型+设计+工程=产品"
- 产品人员参与后训练对话
- 将学习反馈循环回研究

## 反常识视角

### 1. 产品团队的最大价值不在 UX

**传统认知**: 产品团队负责用户体验和界面设计

**现实**: 在 AI 公司，产品团队与研究团队协作产生最大价值，而不是单独构建用户体验

**原因**: 仅使用现成模型可以构建的产品，其他人也能构建。真正的竞争优势在于模型能力和产品体验的"魔法交叉点"

### 2. 代码审查效率悖论

**传统认知**: 代码审查应该逐行进行以确保质量

**现实**: Claude 生成的 PR 往往太大，无法逐行审查

**解决方案**: 用不同的 Claude 审查，人类做验收测试。虽然有利有弊，但目前运作良好

### 3. Consumer Mindshare 不是一切

**挑战**: ChatGPT 在消费者认知中占据主导地位

**Anthropic 的策略**: 拥抱自己是谁，而不是试图成为别人

**聚焦领域**:
- 开发者品牌(Developer brand)
- 创造者品牌(Builder brand)
- 帮助人们完成工作而不是追求病毒式传播

**Mike 的观点**: "可能会有几家代际重要的 AI 公司同时存在。让我们找出我们能 uniquely good at 的东西。"

### 4. Artifact 关闭的反思

**成功指标**:
- 不是追求 Instagram 级别的用户增长
- 但希望有自发生长的势头

**关闭原因**:
1. **移动网站体验恶化**: 发布商的侵入式广告破坏体验
2. **自然传播困难**: 新闻消费高度个人化，不像 Instagram 那样有社交分享动力
3. **远程创业挑战**: 缺乏面对面解决问题的时刻(如在 Burrito 店讨论)

**能量不匹配**: "10 单位输入换 1 单位输出"——投入大量心血但指标几乎不动

**重要教训**: 知道何时停止比知道何时开始同样重要。停止得到支持而非批评，因为这释放了人们去从事更有趣的事情

### 5. AI 创业公司的安全区

**至少 1-3 年内相对安全的领域**:

1. **特定行业知识**: 法律(如 Harvey)、生物科技、医疗保健
2. **差异化市场进入**: 深度了解客户、知道向公司中的谁销售
3. **全新交互范式**: 现有巨头难以适应的全新 UI/UX 方法
4. **创业式工作方式**: 那种"你对抗世界"的存在感和紧迫感

**关键**: "你不能用 OKRs 复制那种感觉，你必须感觉到它。"

Claude 3.7 的反应:

- 声称 Mike 不在 Anthropic 工作

**Claude 给 Mike 的信息**:

### Claude Code 团队

**最前沿的工作方式**:
- 使用 Claude Code 构建 Claude Code
- 自我改进的循环
- 95%+ 的代码由 Claude Code 自己编写
- TypeScript 项目，尽管 Anthropic 主要是 Python/Go/Rust

**有趣的案例**:
- 团队成员不懂 TypeScript，直接与 Claude 对话
- 一小时内提交 PR 解决问题
- 打破了语言选择的障碍

### MCP(Model Context Protocol)的愿景

**核心价值**:
- 不是一次性集成，而是可重复的协议
- 让一次构建的集成在 Claude、ChatGPT、Gemini 等多个平台都能使用
- "商品化你的补充品"(Commoditize your complements)

**未来愿景**:
- Claude AI 的所有构建块(projects, Artifacts, styles 等)都应该暴露为 MCP
- 让一切可脚本化、可组合、可代理化
- Claude 本身应该能够写回到这些原语
