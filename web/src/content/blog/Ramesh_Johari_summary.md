# Ramesh Johari - Podcast Summary

## 嘉宾简介
Ramesh Johari是斯坦福大学教授,专门研究数据科学方法和实践,特别关注在线市场的设计和运营。他曾为Airbnb、Uber、Stripe、Bumble、Stitch Fix、Upwork等世界上最大的市场公司提供建议和合作。

## 核心议题
- 市场平台的真正价值主张
- 数据科学在市场平台中的三个核心循环
- 市场平台的失败模式
- 因果推断vs机器学习
- 实验文化的陷阱
- 评分系统的设计原则
- AI对数据科学的影响

## 核心洞察

### 市场平台的真正价值主张
**"市场平台不是在卖产品,而是在卖移除摩擦"**:
- Airbnb不是在卖房间,房东在卖房间
- Airbnb在卖移除找到住宿的摩擦
- Uber不是在卖乘车,司机在卖乘车
- Uber在卖移除找到司机的摩擦

**交易成本(Transaction Costs)**:
- 在经济学中,这些被称为交易成本
- 市场失败是由于存在这些摩擦
- 市场平台的客户是平台的**双方**:买家和卖家都是客户

### 市场平台数据科学的三个循环
1. **Finding Matches(找到潜在匹配)**:
   - "我想住在某处。谁愿意让我在他们那里住?"
   - "我有房源。谁愿意在我有可用时间时住在那里?"

2. **Making Matches(促成匹配)**:
   - 如果我有多个申请人,我应该雇佣谁?
   - 帮助从可能的匹配中做出匹配

3. **Learning About Matches(学习匹配)**:
   - 评级系统和反馈系统
   - 被动数据收集:你是否在预定离开前离开?
   - 反馈到前两个循环中

### 市场平台最大的失败模式
**在成为市场平台之前就想成为市场平台**:
- "市场平台业务从未作为市场平台业务开始"
- 当你开始时,你没有那种规模的流动性
- 你必须问:"在一个我没有那种规模流动性的世界里,我的价值主张是什么?"

**UrbanSitter的例子**:
- 初始价值主张:接受信用卡支付保姆服务
- 这解决了真正的摩擦:需要现金
- 然后利用Facebook网络建立可信介绍
- 只有在克服了第一个摩擦并获得一些流动性后,才能转向解决市场摩擦

**oDesk(现Upwork)的例子**:
- 初始价值主张:提供工具让工人验证他们在工作时间和做了什么
- 远程工作的信任问题
- 然后提供双方保证
- 在这个阶段,流动性不是游戏

## 心法与原则

### 不要过早称自己为市场平台创始人
**"可能不应该谈论市场平台创始人的概念。真的只有创始人。"**
- 几乎所有人类商业企业都被在线交易的潜力颠覆
- 任何创始人都可能是市场平台创始人
- OpenAI不是市场平台,但现在有插件,变成了市场平台
- Substack开始只是作家的平台,现在投资于为作家驱动需求

### 市场平台管理的打地鼠游戏
**许多最重要的变化创造赢家和输家**:
- 新供应方体验差 → 建立功能定向更有经验的另一方
- 现有方体验变差 → 尝试将他们与更有经验的人匹配
- 指标不断移动

**核心问题**:
- 你创造的赢家是否比你在这个过程中创造的输家对业务更重要?
- 这是一个困难但重要的认知

### 实验不是关于赢和输
**学习是赢**:
- 文化转变:不是"我们测试一切",而是"我们假设驱动"
- 每个测试应该说明关于企业流程、漏斗、偏好的学习
- 失败的实验实际上移动了我们的先验信念

**坏ges(Badges)的例子**:
- 人们认为会很棒的badges实际上很糟糕
- 它们过分关注badged folks,从unbadged folks那里拉走太多注意力
- 如果只以赢/输判断,你会抛弃badges想法
- 但它在教给你关于注意力和库存重新分配的东西

## 实战手册

### 何时真正是市场平台
**规模流动性测试**:
你是否有以下情况:
- 大量买家和大量卖家? → 是市场平台
- 只有买家或只有卖家? → 你赢了一边,现在有选择
- 都没有? → 不要担心成为市场平台,担心扩大一边

**选择点**:
- 如果你有大量买家但没有卖家:如何利用扩大的买家基础吸引卖家?
- Uber的例子:在新城市分发优惠券给免费乘车,补贴司机基础来吸引乘客

### 数据科学的核心区别
**预测vs决策**:
- 机器学习 = 预测 = 相关性
- 决策 = 因果关系

**LTV模型的例子**:
- 预测:这个人有高LTV
- 决策:因为发送这个促销,他们的LTV会增加多少?
- 我们对差异感兴趣,而不是绝对值

**市场平台中的应用**:
- 排名算法:不要只看他们能多好地重现过去的选择
- 看:这个排名算法是否比那个算法导致更多预订/收入?
- 雇佣预测:不要只预测谁会被雇佣
- 看:这个算法是否比那个算法导致更好的匹配?

### 实验文化的危险
**风险规避的两个维度**:
1. **决定测试什么**:
   - 在"测试一切"的文化中,人们倾向于增量测试
   - 更容易获得"赢"
   - 需要尝试风险更大、更有tails的事情

2. **运行实验多久**:
   - 人们倾向于运行实验太长时间
   - 可能比他们应该的更长

**激励机制问题**:
- 数据科学家根据季度"赢"的数量判断
- 更容易通过增量测试获得赢
- 必须运行足够长时间以证明它们是真正的赢
- 不愿意为了尝试风险更大的东西而削减某事

**Bayesian A/B测试**:
- 采用过去的先验信念
- 将实验数据与先验连接
- 奖励为整个企业的先验做出贡献的人
- "你的失败实验实际上移动了我们的先验"

### 评分系统设计原则
**平均化的问题**:
- 默认是平均评分
- 对于有10,000条评论的餐厅,下一条评论无关紧要
- 对于新市场进入者,第一条负面评论可能完全毁了他们
- eBay研究显示:第一条负面评论导致收入立即下降8%

**Prior解决方案**:
- 用先验信念平均,而不只是平均评分
- 先验说:"你得到一个负面评分,但你可能有点倒霉"
- 先验实际上拉高你的评分,给你一个机会
- 分布公平性元素

**评分通胀**:
- 随着时间推移,中位数评分通胀
- 互惠问题:人们不想刻薄
- 规范化:4星感觉像是在伤害某人

**重新规范建议**:
- 不只是"差到优秀"
- 顶级评分实际上是"超出预期"
- "这与你过去评很高的一次体验相比如何?"

### "Learning is costly"(学习有成本)
**未授权holdout group的例子**:
- 营销数据科学经理创建未授权holdout group
- 年底成本:几百万美元
- 理由:#1 现在你知道我的团队值多少
- #2 你永远不会有那个答案,除非我这样做

**教训**:
- 当你不知道答案时,你会给所有选项分配一些样本
- 看着处理后的账单:"为什么要对所有这些食物点那么多?" - Seinfeld片段
- 学习的文化问题:我们使用"赢和输"的语言
- 如果奖励shipping winners,你实际上是在说浪费样本测试失败者是失败

## 反常识视角

### 市场平台不是关于流动性,而是关于价值主张
- 早期阶段,你可能在任何一边都没有流动性
- 那不是市场平台业务
- 专注于解决一个非市场平台问题

### 实验不应该是关于赢
- 科学的实验从未关于赢家
- 它是关于假设
- 每个测试应该说明:我们将学习关于业务流程、漏斗、偏好、需求弹性的什么?

### 机器学习可能不是数据科学的未来
- AI实际上扩大了我们可以考虑的问题前沿
- 假设、解释和原则的天文数字爆炸
- 这对人类施加了更多压力,而不是更少
- 人类在循环中变得更加重要

### 评分系统没有"正确"答案
- Ramesh认为没有人真正"掌握了"这个
- 我们仍在玩eBay和Amazon开始时的相同工具
- 评分系统是研究不足的
- 匿名双盲评分(Airbnb)增加了评分率,但可能没有提高诚实度

### 市场平台的成功可能是短期的失败
- Superhost可能不会在短期内增加预订
- 它重新分配注意力
- 短期可能有赢家和输家
- 但长期可能对平台健康至关重要
