# Nickey Skarstad - 产品愿景与执行专家访谈摘要

## 嘉宾简介
Nickey Skarstad是Duolingo的产品总监,曾在Etsy工作7年从论坛版主晋升到产品总监,在Airbnb领导Experiences团队从0到1找到产品市场匹配,在The Wing担任VP产品,在Shopify从事平台产品工作。

## 核心议题
1. **愿景设定与执行** - 如何设定愿景、转化为目标并执行
2. **产品质量管理** - Airbnb对产品品质的执念和运营方法
3. **产品组织结构** - 功能型vs GM模式的权衡
4. **产品评审流程** - 如何设计产品评审会议

## 核心洞察

### 产品质量的文化建设
- **Airbnb的质量执念** - "Airbnb不发布不是高质量的产品",所有边缘情况都被考虑
- **平衡指标策略** - Experiences用评论率作为顶级指标,平衡增长和质量
- **"First Sale in 7 Days"** - Etsy发现让卖家在7天内获得第一交易是成功的关键质量预测指标
- **增加摩擦以提升质量** - Etsy在onboarding流程中故意增加摩擦,让卖家更认真准备商品,反而提高了成功率

### 愿景到执行的转化框架
- **愿景-使命-策略-目标金字塔** - 从上到下:10年愿景→使命→策略→季度OKR
- **带团队一起** - 不要在真空中做战略,让团队参与创造过程(但不是投票)
- **与领导层对齐** - 确保CEO和创始人buy in,因为他们控制资源
- **OKR的作用** - "好的OKR只是你策略的清晰表达,把重要事情落实为未来3个月的具体行动"

### 二阶决策思维框架
- **定义** - "你能够超越今天的决策,思考今天的决策将如何影响明天的决策和你基于今天决策构建的能力"
- **重要性** - 在市场places和UGC产品中特别重要,因为变更影响整个生态系统
- **操作化方法**:
  1. 在PRD中增加"二阶影响"部分
  2. 写下first principles
  3. 用Miro头脑风暴"这个变更如何影响我们生态系统的各个部分"
  4. 思考:如果我做这个变更,将如何影响明天的决策?

### 一道门vs两道门决策
- **一道门决策** - 难以改变,会限制未来选项,需要花时间讨论、获得反馈和buy-in
- **两道门决策** - 可以改变,给团队自主权,快速移动
- **Experiences质量标准** - 定义"什么是好的体验"的里程碑项目,是典型的一道门决策
- **Etsy可销售品类** - 决定什么可以在市场销售,也是一道门决策

### 产品评审的三检查点
1. **First Principles Check-in** - 要构建什么、解决什么、什么最重要、为谁优化
2. **Approach Check-in** - 如何解决、包含技术组件(基础设施审查、架构审查)
3. **Ready to Ship Check-in** - 是否准备好发布
- **关键** - 第一次对齐最重要,节省团队最多时间

### 远程工作的PM工具
- **异步优先** - 能在Slack解决的就不要开会
- **Loom视频更新** - 5分钟录屏反馈,快速同步领导反馈
- **Huddles** - Slack的音频功能,适合standup
- **白板工具** - Miro, FigJam, MURAL替代面对面白板
- **可见沟通** - 不要依赖Zoom会议填充所有时间

### 产品组织结构选择
- **功能型(Functional)** - 所有人向VP/CPO汇报,适合:
  - 需要大量功能发展的组织
  - 有很多APM或新PM需要支持
  - 不同团队解决相似问题
- **GM型** - 业务负责人管理所有功能,适合:
  - 独特业务机会
  - 需要自主资源和流程
  - Airbnb Experiences案例
- **关键原则** - 没有正确答案,取决于你要构建什么和所处阶段

### 第二阶思维的实际应用
- **Airbnb房源列表** - 修改列表标题长度、数据字段等会影响整个系统
- **Etsy手工定义** - 定义什么算"手工制作"是一阶决策,影响所有列表
- **Shopify的First Principles** - 在变更前写下first principles,帮助后期避免heartache
- **思考框架** - 读《Thinking in Systems》学会二阶效应思维

## 心法与原则

### 关于战略制定
1. **带上团队旅程** - 不要独自写文档然后说"这是我们的战略"
2. **清晰的领导者** - 好的产品工作往往不是民主的,需要一个人负责
3. **收集组织背景** - 与领导层和大业务负责人交谈,理解更广泛背景
4. **CEO的buy in** - 最终,CEO选择资源分配,必须让他们buy in

### 关于决策制定
1. **《Never Split the Difference》** - FBI谈判专家的书,强调同理心和倾听
2. **倾听反对意见** - 理解团队为什么不resonate,可能你是错的
3. **二阶思维是肌肉** - 通过犯错和意识到后果来建立,多练习就会变好
4. **系统思维** - 读《Thinking in Systems》,理解决策如何级联

### 关于产品质量
1. **Dogfooding** - 团队持续使用自己的产品,快速发现问题
2. **质量指标平衡增长** - 不要只看增长,要同时看质量指标
3. **边缘案例思考** - Airbnb对所有边缘情况的执念
4. **用户反馈即时响应** - Brian Chesky的text文化,但要让自主团队感受那种健康的压力

### 关于个人成长
1. **"红黄绿"日历分析** - 给会议着色,看什么给你能量
2. **知道你爱什么** - 优化下一份工作给你爱的东西
3. **理解实际工作内容** - 不同公司和产品的PM日常非常不同
4. **0到1 vs 规模化** - Nickey发现自己更喜欢0到1和产品市场匹配阶段

## 实战手册

### 如何做愿景规划
1. **用FigJam/Miro/MURAL** - 远程白板工具
2. **准备提示词**:
   - 我们10年后在哪里?
   - 未来10年竞争格局会如何变化影响我们?
   - 我们在构建什么?
3. **跨职能头脑风暴** - 邀请营销、政策等其他团队
4. **设置计时器和音乐** - FigJam的音效很好
5. **会议内综合** - 把相似想法归类
6. **不需要在会议中得出最终愿景** - 可以草拟后再反馈

### 如何写OKR
1. **季度周期** - Duolingo用季度OKR
2. **第三个月开始** - 当前季度第三个月开始思考下季度
3. **连接长期计划** - 否则感觉混乱,没有锚点
3. **领导层审查** - 提交给领导层获得批准
4. **清晰表达策略** - OKR是你策略的清晰表达
5. **3-6个月颗粒度** - 让团队能从策略走到feature级别

### 如何做产品评审
1. **三个检查点** - First principles → Approach → Ready to ship
2. **跨职能参与** - 设计、技术、产品一起评审
3. **清晰的预读材料** - 提前发送讨论材料
4. **明确评审标准** - 是否满足目标?是否达到质量标准?
5. **区分一道门和两道门** - 一道门需要充分讨论,两道门给团队自主

### 如何管理远程团队
1. **Loom快速更新** - 领导反馈后5分钟录屏发到Slack
2. **Slack Huddles** - 30秒音频对话,不需要视频
3. **每周团队会议** - 持续检查"我们在尝试做什么?""这些目标如何支持战略?"
4. **异步优先** - 能在Slack做的就不要开会
5. **可见沟通** - 在可见渠道沟通,不要依赖Zoom会议

### 如何知道角色/公司是否适合
1. **"红黄绿"日历分析** - 一周后看日历,红色(无聊/压力),黄色(基线),绿色(兴奋)
2. **问自己**:
   - 我每天实际做什么?
   - 这个工作给我的能量是什么?
   - 我喜欢什么类型的产品工作?
3. **理解组织差异** - 每个公司的产品组织和日常非常不同
4. **优化你爱的工作** - 知道你作为PM最爱的部分,优化下一份工作

### 如何处理冲突
1. **倾听反对意见** - 理解为什么你的战略不resonate
2. **用同理心** - 重复他们说的,让他们感到被heard
3. **明确决策者是PM** - 产品工作往往不是民主的,需要清晰决策者
4. **保持谦逊** - 好的PM是谦逊的人,不总是对的

## 反常识视角

1. **增加摩擦提高成功** - Etsy在卖家onboarding增加摩擦,反而提高成功率
2. **增长不是唯一指标** - Airbnb用评论率平衡增长,质量是顶级指标
3. **功能型vs GM无对错** - 两种结构都可以成功,取决于要构建什么
4. **在真空中做战略** - 很多总监独自写战略文档,但这永远不会好
5. **PM不是民主工作** - 虽然要收集input,但最终需要一个人负责
6. **平台角色可能不适合你** - Nickey发现自己不喜欢平台,即使Shopify是伟大的公司
7. **红黄绿日历揭示真相** - 看一周日历颜色就知道你是否喜欢这份工作
8. **产品评审不需要每周** - 取决于项目,可能需要时才做
9. **远程PM更难** - 所有让PMinform的劳动都更难,但新工具有帮助
10. **听非技术人的播客** - 不要只关注tech Twitter,要了解文化zeitgeist
