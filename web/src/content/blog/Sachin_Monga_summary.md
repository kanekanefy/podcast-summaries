# Sachin Monga - Podcast Summary

## 嘉宾简介

Sachin Monga是Substack产品负责人，此前创办社交应用Cocoon被Substack收购。在Facebook工作7年，领导增长团队、开发者平台、广告增长团队和视频产品团队。

## 核心议题

1. Substack的推荐功能 - 产品史上最传奇的增长功能之一
2. 与产品导向型创始人合作的技巧
3. 从大公司(Facebook)到创业公司(Substack)的适应差异
4. Substack的产品哲学：作者和读者控制权
5. 写作经济的新黄金时代

## 核心洞察

**比喻/英文术语:**

- **黄金时代(Golden Era)** - 互联网写作经济正处于新时代，创作者可通过少量高价值读者谋生
- **一千铁粉(1000 True Fans)** - Kevin Kelly理论，1000个愿意付费的粉丝就能谋生
- **控制权原则(Control Principle)** - Substack核心理念：作者控制内容，读者控制体验，平台不干预
- **你并不晚(You Are Not Late)** - Kevin Kelly文章，互联网才刚刚开始，还有巨大机会
- **供应驱动需求(Supply Drives Demand)** - 作者带来读者，Substack早期专注供应侧
- **良性循环病毒式(Goodwill Viral Loop)** - 推荐功能创建的善意传播：推荐别人→收到通知→感激并反向推荐
- **产品团队时间线(Product Team Timeline)** - 过程一旦完善就过时，因为做好意味着成长，成长意味着旧过程失效
- **单向门vs双向门(One-way vs Two-way Doors)** - 大公司更多不可逆决策，创业公司决策可逆性强
- **读者大脑的写入权(Write Access to Reader's Brain)** - 订阅是重量级行为，作者获得读者深度信任

## 心法与原则

1. **与作者共建(Build with Writers)** - 重大功能改变必须先与作者合作测试，不能直接全量发布
2. **原则优先于增长** - 即使算法推荐更有效，也要坚持作者控制权原则
3. **读者是宽容的** - 读者比作者想象的更宽容，愿意支付、允许休假
4. **永不满足但要适应变化** - 产品团队必须每周每月每年都在进步，但过程会不断变化
5. **信任创始人愿景** - 产品领导者需要追上创始人多年的思考积累
6. **服务永恒问题** - 团队围绕客户问题而非产品功能组织，永远不会"完成"
7. **从小做起** - 写作成功从"开始写"开始，不需要宏大计划

## 实战手册

**与产品导向型创始人合作:**

1. **初期角色定位** - 做促进者而非决策者，确保团队和创始人信息同步
2. **建立节奏** - 每周开始和结束时与创始人对齐重点和担忧
3. **追上愿景** - 努力理解创始人多年的思考积累，帮助团队理解
4. **建立信任** - 通过重复的可靠决策建立信任关系
5. **适应变化** - 公司快速成长意味着流程不断过时，接受这是常态

**Substack推荐功能成功要素:**

1. **作者完全控制** - 作者自己选择推荐谁，不给读者推送不相关内容
2. **出现在订阅流程** - 用户刚订阅一个作者时，最可能接受推荐
3. **创建善意循环** - 被推荐者收到通知，感激并反向推荐
4. **低摩擦操作** - 作者只需选择推荐谁，平台自动处理分发
5. **早期小规模测试** - 与Lenny等10位作者先测试，快速迭代

**在Substack上成功的建议:**

1. **开始就现在** - 注册Substack，写一两篇，看看反应
2. **不要担心太晚** - 总有人说太晚了，但其实机会巨大
3. **1000铁粉足够** - 1000个付费用户就能达到6位数收入
4. **持续发布** - 成功关键是坚持，至少9个月每周发布
5. **不要害怕收费** - 读者比你想的更愿意付费
6. **可以休假** - 读者支持你，允许你休息
7. **利用网络效应** - 推荐功能带来70%新用户

**产品团队组织:**

1. **按客户而非功能组织** - 作者团队、读者团队、增长团队
2. **保持结构稳定** - 不频繁重组，围绕永恒问题而非临时功能
3. **从小到大演进** - 从0个PM到4个PM，从15个工程师到3个全栈团队

## 反常识视角

1. **算法不一定更好** - Substack拒绝"推荐算法"，坚持作者人工推荐，效果反而更好
2. **创始人最初反对最佳功能** - Chris最初怀疑推荐功能可行，认为步骤太多
3. **读者比作者想象的更宽容** - 作者担心收费、休假会惹恼读者，其实读者很支持
4. **大公司经验未必有用** - Facebook的产品管理经验在Substack大部分不适用
5. **过程完善=过程过时** - 把产品流程做好就意味着它要过时了，因为公司成长了
6. **推荐功能只需简单实现** - 不需要AI推荐，作者人工选择足够了
7. **付费用户意向不低** - Lenny 70%增长来自推荐，但打开率只下降一点点
8. **早期不要专注读者侧** - Substack先专注作者，因为供应驱动需求
