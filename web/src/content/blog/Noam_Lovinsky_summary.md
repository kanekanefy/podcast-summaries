# Noam Lovinsky - Lenny's Podcast 播客摘要

## 嘉宾简介

Noam Lovinsky 是 Grammarly 的首席产品官（CPO）。他有着非常独特的职业轨迹：

- **YouTube 早期 PM**：花费5年时间领导创作者产品体验，然后领导更广泛的 YouTube 消费者产品体验
- **Thumbtack CPO**：帮助公司在 Google SEO 变化导致的衰退后重新点燃增长
- **Facebook**：创建了新产品实验（New Product Experimentation）团队，其章程是在不受大型 Facebook 组织干扰的情况下孵化大型新想法
- **Grammarly CPO**：现任职位

Noam 从未上过播客，也很少在 Twitter 上发布内容或在线发布任何东西。他在将产品从0到1、从-1到1、从1到100方面有着独特的经验，甚至创办了自己的公司。

## 核心议题

本次对话涵盖了 Noam 在 YouTube、Facebook、Thumbtack 和 Grammarly 的职业生涯中学到的经验教训：

1. **何时杀死你的项目** - 在公司内部
2. **何时要求被"分层"（layered）** - 在公司内部
3. **为什么你应该关注哪些产品对业务最重要** - 以及如何找到这些产品
4. **为什么要多元化你的增长渠道** - 在业务中
5. **为什么应该找到最能拉伸你的工作** - 帮助你在职业中前进
6. **在大公司内部为创新创造空间** - 的大量建议
7. **从 -1 到 1 的经验** - 帮助 Thumbtack 从衰退中恢复
8. **新产品实验团队的创建** - 在 Facebook

## 核心洞察

### 1. 产品生命周期管理

Noam 经历了产品的不同阶段：
- **-1 到 1**：Thumbtack 的恢复增长
- **0 到 1**：新产品创建
- **1 到 100**：YouTube 的扩展

**关键洞察**：每个阶段需要不同的技能和思维模式。

### 2. 何时杀死项目

Noam 分享了在公司内部何时应该杀死项目的标准。

### 3. 何时要求被"分层"

"分层"（layered）意味着在组织结构中，有人被放在你之上。Noam 讨论了何时这是一个正确的职业举动。

### 4. 产品重要性的识别

**为什么重要**：
- 不是所有产品对业务的贡献都相等
- 需要识别哪些产品真正驱动业务价值
- 将精力和资源集中在正确的地方

**如何找到**：
- 分析业务指标
- 理解收入和增长驱动因素
- 与公司战略对齐

### 5. 增长渠道多元化

**Thumbtack 的经验教训**：
- Google SEO 的变化严重打击了 Thumbtack
- 过度依赖单一增长渠道是危险的
- 必须多元化增长渠道以降低风险

**启示**：
- 不要把所有鸡蛋放在一个篮子里
- 主动投资多个增长渠道
- 建立抗风险能力

### 6. 职业发展的拉伸原则

**核心理念**：
- 应该找到最能"拉伸"（stretch）你的工作
- 舒适区不会带来成长
- 寻找稍微超出你当前能力的挑战

**实践**：
- Noam 的职业选择都是基于学习机会
- 从 YouTube 到 Thumbtack（衰退中的公司）到 Facebook（新产品团队）
- 每次选择都是因为能学到新东西

### 7. 大公司内部创新

**Facebook NPE 团队**：
- 章程是孵化大型新想法
- 受保护，不受大型 Facebook 组织干扰
- 给予创新空间和自由

**关键要素**：
- 独立性
- 资源
- 时间
- 免受核心业务压力

## 心法与原则

### 核心心法

1. **成长来自不适**
   - 寻找拉伸你的机会
   - 舒适区是成长的天敌
   - 主动选择挑战性工作

2. **产品重要性不平等**
   - 并非所有产品都同等重要
   - 需要识别真正的驱动因素
   - 将资源集中在最重要的地方

3. **多元化是必要的**
   - Thumbtack 因过度依赖 Google SEO 而受损
   - 必须建立多个增长渠道
   - 风险管理是产品领导力的一部分

4. **创新需要保护**
   - NPE 团队需要在 Facebook 内部受到保护
   - 大型组织会压倒创新努力
   - 必须创建创新空间

### 关键原则

1. **诚实评估**
   - 诚实面对项目是否应该继续
   - 不要因为情感投入而坚持失败的项目
   - 杀死项目可能是正确的事情

2. **职业战略**
   - 有意识地选择能最大化学习的工作
   - 不只是追求头衔或薪酬
   - 关注长期成长而非短期收益

3. **保护创新**
   - 在大公司内部，创新需要特殊保护
   - 创建独立的团队和流程
   - 给予时间和空间实验

## 实战手册

### 如何识别重要产品

#### 步骤：

1. **分析业务指标**
   - 收入贡献
   - 用户增长
   - 留存率
   - 利润率

2. **理解战略重要性**
   - 是否支持公司战略？
   - 是否是差异化因素？
   - 是否有长期潜力？

3. **评估资源使用**
   - 投入了多少资源？
   - 产出是否匹配投入？
   - 机会成本是什么？

4. **识别依赖关系**
   - 其他产品是否依赖它？
   - 它是否是平台产品？
   - 是否有网络效应？

### 如何多元化增长渠道

#### Thumbtack 的教训：

1. **承认风险**
   - Thumbtack 过度依赖 Google SEO
   - Google 的算法变化打击了他们
   - 必须认识到这种依赖的危险

2. **投资多个渠道**
   - 不要依赖单一渠道
   - 测试和投资多个增长渠道
   - 平衡短期和长期渠道

3. **建立自有渠道**
   - 减少对第三方平台的依赖
   - 投资于直接用户关系
   - 建立品牌忠诚度

4. **持续实验**
   - 持续测试新渠道
   - 快速学习和调整
   - 不要固守成功的历史渠道

### 何时杀死项目

#### 评估标准：

1. **战略对齐**
   - 项目是否仍然与公司战略对齐？
   - 是否支持核心业务目标？

2. **资源效率**
   - 投入是否值得产出？
   - 资源是否可以更好地用在别处？

3. **市场反馈**
   - 用户是否真的想要这个？
   - 是否有产品市场契合度？

4. **团队士气**
   - 团队是否仍然相信？
   - 是否有执行的能量？

**决策过程**：
- 诚实评估
- 收集数据
- 征求多方意见
- 做出艰难决定

### 何时要求被"分层"

#### 考虑因素：

1. **学习机会**
   - 是否能从新领导身上学习？
   - 是否能获得新视角？

2. **项目成功**
   - 这是否有利于项目成功？
   - 是否能带来额外资源？

3. **职业发展**
   - 是否能帮助你成长？
   - 是否能打开新机会？

4. **公司需求**
   - 这是否是公司需要的？
   - 是否符合组织结构？

### 如何在大公司内部创新

#### NPE 团队模式：

1. **独立性**
   - 独立于核心业务
   - 自己的文化和流程
   - 独立的决策权

2. **资源**
   - 专用预算
   - 专门人才
   - 时间承诺

3. **保护**
   - 免受短期压力
   - 不被核心业务指标衡量
   - 允许失败

4. **对齐**
   - 与公司战略对齐
   - 但有独立的指标
   - 清晰的章程

## 反常识视角

### 1. 选择衰退中的公司

**反常识**：通常人们会选择增长中的公司

**Noam 的选择**：
- 从 YouTube（成功中）到 Thumbtack（衰退中）
- 因为这提供了更多学习和影响力机会
- 能够帮助公司恢复增长是宝贵的经验

**启示**：
- 不要只追求表面的成功
- 衰退中的公司可能提供更多学习机会
- 你可以真正产生影响

### 2. 要求被"分层"是好的

**反常识**：通常认为被分层是负面的

**Noam 的观点**：
- 有时候要求被分层是正确的
- 如果这能带来更好的学习机会
- 如果这能帮助项目成功

**启示**：
- 不要只看头衔和层级
- 关注学习和发展机会
- 做出对长期职业发展有利的决定

### 3. 杀死项目是正确的

**反常识**：通常认为坚持是美德

**Noam 的观点**：
- 有时候杀死项目是正确的
- 需要诚实面对现实
- 资源应该用于更好的机会

**启示**：
- 不要因为情感投入而盲目坚持
- 诚实评估项目前景
- 勇敢做出艰难决定
