# Paige Costello 播客摘要

## 嘉宾简介

Paige Costello 是 Asana 的产品领导，从Asana早期加入并成长为产品主管，领导了多个核心功能的开发，擅长产品教练、设计思维和团队协作。

## 核心议题

- 双钻设计流程(Double Diamond Process)的实际应用
- 产品经理的教练技能和反馈文化
- 作为房间里最年轻的人如何建立信任和影响力
- Asana的产品演进和核心价值设计
- 讲故事在产品管理中的重要性
- 新产品经理的最佳实践

## 核心洞察

### 英文术语与比喻

- **Double Diamond Process（双钻流程）**: 设计思维的经典框架，通过两次发散-收敛循环解决问题：先发散理解问题，收敛定义核心问题，再发散探索解决方案，最后收敛选择最佳方案

- **Trust Equation（信任公式）**: 建立信任需要四个要素：可信度(competence)、可靠性(reliability)、亲密感(intimacy)、自我取向(self-orientation，越低越好)

- **How might the opposite be true?（反面可能怎样?）**: 一个强大的质疑假设工具，帮助团队摆脱惯性思维，考虑完全相反的可能性

- **Onboarding as a Product（入职即产品）**: 把员工入职体验当作产品来设计，有清晰的用户旅程、成功指标和持续优化

### 关键洞察

1. **双钻的实践价值**: "双钻流程不是理论框架，而是实际工作方法。它帮助我们避免急于解决问题，先确保我们解决的是正确的问题"

2. **反馈是礼物**: "最好的团队文化是人们主动寻求反馈，并且把反馈当作礼物而不是攻击"

3. **信任是基础**: "作为产品经理，如果你没有信任，再好的想法也卖不出去。信任方程提醒我需要降低自我取向，真正关心团队"

4. **质疑假设**: "我们总是陷入惯性思维。问'反面可能怎样?'能帮助我们打破思维定式，发现更好的方案"

5. **入职体验的重要性**: "很多公司忽视了入职体验，但这是影响员工长期表现的关键时期。应该像对待产品一样对待入职"

## 心法与原则

1. **先理解问题，再解决问题**
   - 双钻的第一钻是理解问题
   - 不要急于跳到解决方案
   - 投资时间在问题定义上

2. **主动寻求反馈**
   - 把反馈当作成长的工具
   - 创造安全的反馈环境
   - 定期请求具体反馈

3. **建立信任胜过展示聪明**
   - 可信度来自能力展示
   - 可靠性来自言行一致
   - 亲密感来自真诚关心
   - 低自我取向是关键

4. **讲故事的力量**
   - 人们记住故事胜过数据
   - 好的故事传递情感和信息
   - 产品经理必须成为好讲者

5. **持续优化入门体验**
   - 无论是用户还是员工
   - 第一印象决定长期关系
   - 把入职当作产品设计

## 实战手册

### 应用双钻流程

**第一钻:问题空间**

1. **发散阶段**
   - 广泛收集用户研究
   - 访谈多个利益相关方
   - 观察用户实际行为
   - 收集定量和定性数据

2. **收敛阶段**
   - 识别核心问题
   - 排除表面问题
   - 聚焦真正需要解决的问题
   - 获得团队对问题定义的共识

**第二钻:解决方案空间**

3. **发散阶段**
   - 头脑风暴多个方案
   - 鼓励疯狂想法
   - 暂时不做评判
   - 探索各种可能性

4. **收敛阶段**
   - 评估和筛选想法
   - 原型测试
   - 选择最佳方案
   - 制定实施计划

### 建立信任的实践

1. **展示能力**
   - 准备充分的会议
   - 提供有深度的分析
   - 展现专业知识
   - 承诺并交付高质量工作

2. **建立可靠性**
   - 言行一致
   - 按时完成承诺
   - 及时沟通问题
   - 保持可预测性

3. **创造亲密感**
   - 真诚关心他人
   - 分享个人故事
   - 展示脆弱性
   - 倾听和理解

4. **降低自我取向**
   - 关注团队成功而非个人
   - 承认错误和不足
   - 把他人利益放在首位
   - 寻求共同胜利

### 作为年轻产品经理的影响策略

1. **准备充分**
   - 研究会议参与者
   - 准备数据支撑观点
   - 预想可能的问题
   - 带来洞察而非仅仅是信息

2. **建立盟友**
   - 寻找支持你的资深同事
   - 在会前获得关键人物的认同
   - 利用一对一关系建立支持

3. **聚焦价值**
   - 强调对用户和业务的价值
   - 用数据和事实说话
   - 展示准备和研究
   - 证明你为团队利益考虑

4. **承认学习态度**
   - 诚实地表达你想要学习
   - 寻求指导和反馈
   - 展示谦逊和好奇心
   - 不要假装知道一切

## 反常识视角

1. **年轻可以是优势**: "作为房间里最年轻的人，你有独特的视角。你看到的是事物的本来面目，而不是'我们一直这么做'的思维定式"

2. **反馈不是攻击**: "如果我们改变对反馈的看法，把它当作礼物而非攻击，团队文化会完全不同。主动寻求反馈是成长的捷径"

3. **慢即是快**: "双钻流程看起来很慢，但实际节省时间。如果我们先花时间确保解决正确的问题，后面的工作会更高效"

4. **故事胜过数据**: "人们可能不记得你的数据，但他们会记住你的故事。好的产品经理知道如何用故事传递洞察"

5. **入职最关键**: "公司在产品入职上投入巨大，但在员工入职上投入很少。其实应该把员工入职当作产品来设计"

## 花絮与细节

- Paige在Asana从早期阶段就加入，见证了公司从几十人到数千人的成长历程

- 作为团队中较年轻的成员，她学会了如何在没有正式权威的情况下建立影响力

- 她特别强调双钻流程不是理论，而是她在Asana每天使用的实际工作方法

- "How might the opposite be true?"成为了她团队中常用的质疑工具，帮助他们避免群体思维

- 她认为信任公式是产品领导力的核心：可信度+可靠性+亲密感除以自我取向

- Asana的产品文化强调把员工入职当作产品来设计，有清晰的成功指标和持续优化流程

- 她观察到最好的产品经理不是最聪明的人，而是最能建立信任和影响力的人

- 她鼓励年轻产品经理不要假装知道一切，承认学习态度反而会赢得尊重
