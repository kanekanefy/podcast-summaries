# Scott Belsky - Podcast Summary

## 嘉宾简介
Scott Belsky是Adobe首席战略官兼设计、新兴产品负责人,前首席产品官。Behance创始人(被Adobe收购),畅销书《The Messy Middle》作者,知名天使投资人和产品思想领袖。

## 核心议题

1. **产品感(Product Sense)的本质**:如何培养和发展产品直觉
2. **"第一英里" (First Mile) 体验**:为什么onboarding是最大机会
3. **AI时代的产品工作**:AI如何改变产品开发和PM角色
4. **The Messy Middle**:如何在创业和产品开发的中期坚持下去
5. **资源vs资源禀赋**:为什么限制是创新的催化剂

## 核心洞察

### 第一英里心理学:前30秒决定成败
- **人类本性**:新产品前30秒,用户是lazy, vain, selfish的
  - Lazy:想要快速完成
  - Vain:想要在朋友/同事面前看起来不错
  - Selfish:想要快速感受成功
- **最大错误**:团队把90%时间花在"最后一英里",只10%在"第一英里"
- **反直觉**:改善onboarding比优化功能影响大10倍
- **持续优化**:每个新客户群体都需要重新思考onboarding

### 产品感的金标准:共情胜于激情
- **最大陷阱**:团队对solution充满激情,但对客户problem缺乏共情
- **正确方式**:深入理解客户痛苦 → 解决方案自然显现
- **实战方法**:
  - Shoulder to shoulder观察客户日常
  - 理解他们使用产品时的context(开会、带娃、多任务)
  - 感受他们的friction和pain points
- **"为他人而做"**:通过共情变成客户 → 最佳产品诞生于"maker是customer"

### 惊喜与愉悦:口碑传播的引擎
- **反常识**:人们不会谈论"产品达到预期",只会谈论"意外惊喜"
- **案例**:Tesla不会谈论"今天开得很好",但会讨论dashboard的Easter egg
- **实战启示**:
  - 优化那些超出预期的时刻
  - 创造magic trick般的体验
  - 让用户发现惊喜,而非罗列功能
- **好莱坞技巧**:mystery + intrigue + surprise = 口碑传播

### "做一半"哲学(The Half Philosophy)
- **核心原则**:只做你想做的一半功能、一半选项、一半市场
- **学习过程**:Behance早期做了太多功能 → killing spree → 核心指标10x增长
- **决策框架**:
  - Optimize for problems you WANT to have
  - 先消除brick walls(致命障碍)
  - 延迟nice-to-have功能
- **经验法则**:当你在犹豫是否做某功能时,答案通常是"不做"

### AI从工作流到心流(Workflow to Flow)
- **核心转变**:AI消除重复劳动,让人类专注创意和高价值工作
- **乐观视角**:人类生产力上升 → 公司想要更多人类 → 更多产品被创造
- **AI vs人类**:
  - AI: "what it would look like if"(挖掘中心)
  - Human: 寻找become center的edges
- **组织影响**:
  - Collapse the stack(消除中间层)
  - 赋能个人cross-functional工作
  - 数据分析无需依赖data scientist

### 黄金直觉(Golden Gut)
- **定义**:经验丰富的产品决策者在micro-moment的直觉判断
- **典型场景**:
  - "这个步骤要不要简化成一个默认值?"
  - "10%人困惑 vs 90%人更快通过,哪个更好?"
  - "要不要完全去掉这个步骤?"
- **培养方式**:
  - Designer参与upstream讨论
  - 积累大量A/B测试经验
  - 持续观察用户行为

### 坚持还是放弃:信念测试(Conviction Test)
- **问题**:如果你现在知道你所知道的一切,你还会开始这个项目吗?
- **回答A**: "我有更多信念!" → You're in the messy middle,坚持下去
- **回答B**: "如果当时知道这些,我根本不会做" → Quit, pivot,做点别的
- **核心判断**:信念是持续的唯一燃料

## 心法与原则

1. **产品即心理学**
   - 理解人类primal tendencies(懒惰、虚荣、自私)
   - 用psychology设计体验,而非feature list

2. **资源禀赋 > 资源**
   - 如果资源是carbs,resourcefulness是muscle
   - 限制催生创新和创新
   - 最佳公司诞生在寒冬

3. **极简即力量**
   - 做一半你想做的事
   - 每次添加时考虑replace/remove
   - 核心指标>功能数量

4. **体验经济长期看涨**
   - AI解放人类 → 更多人在experience领域工作
   - 非scalable的东西变得更值钱
   - human touch变成竞争优势

5. **例外即规则**
   - 真正transformative的事物都是exception
   - 如果所有人都说你疯了,你要么疯了要么onto something
   - best practices是起点,不是终点

## 实战手册

### 培养产品感
1. **深度共情**
   - 每月做2-3次customer visit
   - Shoulder to shoulder观察完整一天
   - 记录他们使用产品时的context

2. **第一英里优化**
   - 录制新用户onboarding session
   - 识别前30秒的friction points
   - 测试多个简化版本

3. **惊喜时刻审计**
   - 列出产品中可能超出预期的点
   - 对比竞品的surprise elements
   - 设计3个Easter eggs或magic moments

4. **做一半实验**
   - 列出所有计划功能
   - 划掉一半(假设不做)
   - 观察核心指标是否提升

### AI时代PM技能升级
1. **Play强制**
   - 每周尝试新AI工具
   - 用AI做prototype和探索
   - 写关于AI implications

2. **Collapse the stack**
   - 学习用AI做数据分析
   - 用AI生成design options
   - 减少对specialists的依赖

3. **培养golden gut**
   - 让designer参与upstream讨论
   - 积累更多A/B测试经验
   - 记录并反思你的product决策

4. **探索vs执行**
   - 用AI探索10X surface area
   - 自己做reduction到2-3个选项
   - 保持人类taste和judgment

### 判断是否坚持
1. **信念测试**
   - "如果现在重新开始,我还会做这个吗?"
   - 信念下降 → 考虑pivot
   - 信念上升 → 继续坚持

2. **微目标管理**
   - 设定团队认同的小里程碑
   - 定期merchandise progress
   - 创造短期win维持士气

3. **文化作为燃料**
   - 服务客户的使命感
   - 与优秀团队共事的荣誉感
   - 这两个足够支撑度过messy middle

### 资源约束下创新
1. **Refactor > Hire**
   - 用AI自动化而非招人
   - 重构流程而非扩大团队
   - 技术升级而非人力堆叠

2. **Opportunity mindset**
   - 每个限制都是创新机会
   - "如何用现有资源做到?"
   - 强迫找到更efficient方法

## 反常识视角

1. **功能越少,核心指标越高**
   - Behance killing spree案例
   - 移除功能反而提升关键行为
   - Focus creates velocity

2. **AI会创造更多就业**
   - 生产力上升 → 需求更多生产力
   - 公司从3产品→7产品
   - Human ingenuity变得更有价值

3. **Onboarding永远是最大机会**
   - 即使成熟产品也需要持续优化
   - 新客户群体不同,需要重新设计
   - 第一英里改善>最后一英里优化

4. **优秀创始人不是promoters**
   - 提防过度sugarcoat的创始人
   - 看重"optimistic about future, pragmatic about present"
   - 透明度=partnership信任

5. **经验主义vs直觉主义**
   - 数据很重要,但golden gut是差异化
   - AI可以探索更多可能,但人类做最终判断
   - Taste和judgment变得更有价值

6. **PMF不是终点**
   - 每个新segment都没有PMF
   - 产品需要持续re-earn PMF
   - 市场变化让PMF失效

## 花絮与细节

- **Adobe角色演变**:CPO → CSO + Design + Emerging Products,因为边界本身是人为的
- **Implications newsletter**:强制自己理解AI的monthly exercise
- **写作时间管理**:早晨跑步后40分钟ruthlessly保护写作时间
- **育儿哲学**:培养passion muscle memory,具体技能不重要
- **推荐产品**:Queue(跨平台内容队列),Tome(AI生成演示文稿)
- **推荐书籍**:
  - *Build* by Tony Fadell (产品实践智慧)
  - *The 48 Laws of Power* (理解人性)
- **宇宙纪录片爱好**:在downtime看cosmos和black holes纪录片
- **投资哲学**:product design-oriented teams making things that need to exist
