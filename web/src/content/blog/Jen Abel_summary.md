# Jen Abel - 创始人主导销售的实战指南

**嘉宾简介**: Jen Abel - Jellyfish联合创始人,帮助早期创始人学习销售、客户发现和建立可重复销售流程,曾任The Muse和General Assembly的企业销售总监。

## 1. 核心议题 (The Questions)

- **为什么创始人必须亲自做早期销售?** - 创始人是产品,拥有独特洞察,能发现销售中萌发的关键时刻
- **如何找到第一批销售线索并让他们愿意与你对话?** - 通过反向直觉的洞察而非"更好"的推销
- **第一次销售电话应该说什么、做什么?** - 展现脆弱性,收集研究而非立即成交
- **如何穿越企业采购的复杂流程完成签约?** - 了解签字人、简化流程、为采购人员代劳

## 2. 核心洞察 (Core Conclusions)

### 颠覆性认知
- **"0到1的销售人才不存在"** - 这是Jellyfish冷邮件的钩子,创始人必须自己掌握销售
- **创始人销售不是为了第一天收入** - "是为了以人类最快的速度学习,从而获得销售的权利"
- **"销售不是为了收入,是为了收集研究"** - Founder-led sales是收集研究,Sales for revenue是后期阶段
- **40-50%的B2B SaaS公司需要先卖服务才能卖技术** - 用户和买家不同时,必须教育市场

### 关键比喻
- **传声筒游戏** - 如果销售人员说"没人在乎这个",创始人第一反应是"销售的问题",不是"我的愿景问题"
- **留下渴望** - Demo是"你唯一控制的诱惑",不要展示全部,像给投资人pitch一样保持神秘感
- **变成合作伙伴** - 进入企业后"不再是销售,而是伙伴",可以参加会议,要求介绍

### 数据支撑
- **健康的外联转化率**: 5-7%(成熟期),2-3%(早期),8-15%(创始人+痛点问题)
- **销售周期**: 企业销售通常6-12个月,受监管行业9-12个月,90天完成也有可能
- **理想ACV**: 50-200K,早期公司50-100K起步
- **创始人销售里程碑**: 约500K-1M ARR,快则500K可过渡,慢则需继续

### PMF洞察
- **Vanta/Airtable对比** - Vanta从市场痛点开始(快速PMF但有上限),Airtable从技术洞察开始(慢PMF但无上限)
- **从市场开始风险更低,从产品开始上限更高**
- **"如果你找到了,那是巨大的胜利"** - 投资人愿为高技术洞察风险下注

## 3. 心法与原则

### 底层哲学
- **创始人就是产品** - "你是某个高度特定领域的主题专家,你研究、经历了市场大多数人甚至没机会看到或想象的东西"
- **脆弱性创造真实反馈** - 告诉他们你是早期公司,还没完全建好,即使建好了也别这么说
- **信任是销售的第一货币** - "我不认为我是合适的选择"时,客户反而会试图说服你

### 销售六阶段
1. **Intro call** - 介绍电话
2. **Demo call** - 演示电话(根据市场阶段可能合并到第二通)
3. **Proposal call** - 提案/工作范围电话,深入演示结合所学
4. **Feedback call** - 反馈并共同起草工作范围
5. **Procurement intro** - 采购介绍
6. **Signature** - 签约(可能来自CFO、CLO、业务单元负责人、采购负责人)

### 反常识原则
- **第一通电话不要Demo** - 让他们在脑海中想象,不要过早展示
- **别问"什么让你睡不着?"** - 答案每天都在变
- **不要用"更好"推销** - 采购负责人问"怎么定义?怎么衡量?要不给现有公司一年改进?"
- **服务收入有价值** - 展示意图,获得logo,付费教育买家,设计流程

## 4. 实战手册

### 找到30个潜在客户
- **手动列表** - 在用工具前,能否找到30个你愿意花15-20分钟写信的人?
- **发现难度测试** - 如果30个都很难找到,你已经麻烦了
- **共同参数收集** - 团队规模、行业、任期、前任职位等模式
- **多渠道触达** - 邮件、LinkedIn、电话、Twitter DM

### 冷邮件四要素
1. **相关性** - "为什么这对我是相关的?"比个性化更重要
2. **反直觉洞察** - 让他们说"什么?"或"我从没这样想过"
3. **聚焦问题** - 不谈解决方案,只谈需要解决的问题
4. **手机端不用滚动** - 3-4句话最多,留给他们想要更多

### 邮件示例
```
主题:0到1销售人才不存在(原因)

Hi [名字],

我注意到你们在寻找[目标]基于网站信息。

Jellyfish的信念是0到1销售人才不存在。所以我们建立了这个模式。

希望聊聊。
```

### 第一通电话脚本
```
"嗨,我是早期创业公司。我们对解决这个特定问题充满激情。
我们还有很多要学习的。以下是我们从问题优先级角度的思考。
能否让我们获得关于这个问题在你那边如何表现的洞察?"
```

### 问题信号检测
- **问题是否在增长扩大?** - 没人会花时间修复固定问题
- **你们在测量/管理这个问题吗?** - 如果否,可能没什么
- **你们如何尝试解决?** - 招聘人头、工具、仍然是空白?
- **好的信号**: 他们要带同事进下一通电话

### 结束电话
- **当场预订第二通** - 拉开日历,邀请其他人
- **"我会邮件你"的信号** - 90%情况是"我没勇气说不感兴趣"
- **保持通话紧凑** - 不要延长销售周期,两周太长,一周更好

### 共创工作范围
- **90天服务合同** - 时间盒,之后重新评估范围
- **付费教育** - 传统行业例子:"能否来解释如何整合这个,考虑技术前?"
- **设计流程** - AI需要人机策略、成功指标、风险意识

### 穿越采购
- **推销给采购** - 他们是专业买家,很聪明
- **不要复杂化** - 不用术语,让他们容易理解
- **必须感觉不同** - 否则他们会建议用现有17个优选供应商
- **代劳工作** - "我要让这事尽可能简单,给我表格我来填"
- **明确你做什么/不做什么** - 避免被分类为高风险(要求500万保险等)
- **分割合同** - 技术合同+服务合同,服务合同先启动创造激励

### 签约前
- **知道签字人** - CFO/CLO/业务单元负责人/采购负责人
- **为签字人准备简报** - 避免CFO问"我在签什么?"导致回到队尾
- **财务批准前别工作** - 业务单元不能直接付款,需要采购订单
- **折扣策略** - 有理由时给(设计伙伴、两年推荐),不是成交策略

### 工具使用建议
- **先别用工具** - 手动找到30人,写定制邮件
- **参数后用Clay** - 知道正确问题后,才能有效丰富数据
- **避免批量轰炸** - 一天发5000封会影响域名健康

## 5. 反常识视角

### 早期vs晚期销售
- **早期**: 脆弱、学习、共创、服务优先
- **晚期**: 教育市场、展示成熟度、降低购买风险(April Dunford建议)

### 企业销售动力
- **进去后就进去了** - 成为优选供应商,跨业务单元,竞争对手暂时被挡住
- **复利效应** - 10万变成下年50万,50万变后年100万
- **销售努力回报指数级** - "如果付出销售努力,回报是疯狂的"

### 转化率迷思
- **转化率受胜率影响** - 30-40%胜率不需要高转化,10%胜率需要高转化
- **"转化率2% vs 12%"** - 同样外联,唯一差异是创始人对市场的洞察
- **"我们总是有销售问题"** - 大多数时候是"没说什么有趣的",不是销售技术

### 中市场陷阱
- **最难起步的地方** - 夹在两种不同go-to-market之间(高价值vs高量)
- **集成困难** - 资源不足,外包给埃森哲等第三方,变得混乱
- **要么上要么下** - 小企业(高流失但快速)vs企业(慢但粘性)

### 定价心理学
- **"最好"的敌人** - "我会用不够好的方案,因为有太多其他事要做"
- **折扣对话** - "如果我给30%折扣,我能去掉30%价值吗?"
- **小企业缓冲** - 可能是他们的钱,留点折扣空间

## 6. 花絮与细节

### Wiz创始人案例
- 每天10-15通对话持续数周
- 人们说"我喜欢"但不买,不兴奋到想前进
- **只有创始人在这些电话上才能发现**

### Zip 23亿估值案例
- 在LinkedIn联系采购负责人
- **"我们只是想要建议"** - 建议导向vs销售导向
- 倾向你说的"获得洞察"

### Sprig投资人要求
- First Round合伙人:"不到1M ARR不让雇销售人员"
- **"我们会帮你,有销售人协助,但不让你雇人"**

### April Dunford对比
- **她专注晚期公司** - 买软件比卖软件难
- **她的建议**: 教育买家市场走向,这是未来
- **早期相反**: 别装专家,承认早期,收集反馈

### 成功信号
- 他们带其他同事进电话
- 他们问"谁能解决这个问题?"
- 他们承认"去年雇了人管理这个"
- 他们说"我们尝试解决但还没解决"

### 失败信号
- "我会邮件你" - 90%没兴趣
- 问题不在增长/扩大
- 没在测量/管理
- 第一通电话后无反馈

### 采购专业度
- **他们是专业买家** - 每天做这个
- **小采购容易被边缘化** - 你是大交易的一小部分
- **建议用现有供应商** - 如果你不显得不同

### 签约人类型
- **CFO** - 关注财务影响
- **CLO** - 法律风险
- **业务单元负责人** - 业务价值
- **采购负责人** - 流程合规

### 合同陷阱
- **不超过现有收入20%** - 某些企业合同规定
- **可协商性检查** - 问是硬性规则还是可谈
- **10万变2万的风险** - 过度降价可能伤害价值主张

### 推荐资源
- **April Dunford书籍** - 晚期公司定位和销售
- **Lenny的PMF研究** - 创业公司达到PMF时间线图

### Jellyfish服务模式
- **嵌入团队** - 不外包销售核心
- **创始人仍是矛尖** - 直接接触市场学习
- **避免传声筒** - 创始人直接从市场获得反馈

### 个人哲学
- **"帮助彼此"** - 科技界的forward pay模型美丽
- **"销售应该对买家有趣"** - 他们应该觉得"这个人改变我的世界观"
- **"信任是第一货币"** - 受信任的创始人获得持续推荐

### 行业观察
- **AI公司服务合同** - 一两年前纷纷服务合约,设定买家思维
- **传统行业教育** - "5-6年没换供应商,不知道怎么做"
- **监管行业加时** - 20-30%更长时间

### 错误案例
- **Jen的CFO事件** - 没提前知道签字人,CFO问"我在签什么?",延长一个月
- **工具过早使用** - Clay轰炸5000人,域名健康受损
- **Demo过早展示** - "他们眼睛里梦想消失"

### 冷邮件Gmail技巧
- **高亮并朗读** - Gmail可以语音播放邮件
- **Jen的经历** - "多少次我改成,哦,听起来很被动攻击"
