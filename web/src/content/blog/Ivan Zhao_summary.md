# Ivan Zhao - Notion联合创始人兼CEO

## 嘉宾简介
Ivan Zhao - Notion联合创始人兼CEO，从中国乌鲁木齐（400万人口城市）通过编程竞赛第二名进入北京名校，16岁移民加拿大，通过看《海绵宝宝》学习英语文化

## 1. 核心议题 (The Questions)
- **如何在3-4年"迷失岁月"中坚持并最终找到产品市场契合度？** 什么样的心理韧性和战略调整让团队能够度过长期低谷？
- **如何平衡"为世界创造价值"的理想主义与"用户真正需要"的现实需求？** "糖衣西兰花"策略的本质是什么？
- **如何保持极简团队和盈利状态实现百亿估值？** 在不牺牲质量的前提下实现高效的运营模式？
- **横向产品的"快乐与痛苦"是什么？** 如何在一个需要多年才能成熟的品类中坚持并找到成功路径？
- **如何让"乐高式软件"这个抽象概念成为现实？** 如何避免非乐高式硬编码的诱惑？

## 2. 核心洞察 (Core Conclusions)
- **前3-4年是"迷失岁月"**：最初版本是"开发者工具"，试图让每个人都能创建软件，但意识到"大多数人根本不在乎"
- **"糖衣西兰花"(Sugar-coated Broccoli)策略**：人们不喜欢吃西兰花（开发软件的能力），但喜欢糖（生产力工具），所以给他们糖，把西兰花藏在里面
- **技术债务迫使完全重构**：早期押注Google的Web Components技术，发现太不稳定，"不知道bug来自你的代码还是底层库"，不得不放弃整个代码库并重建
- **史蒂夫·乔布斯的误解**：乔布斯参观Xerox PARC看到Alto系统时，只看到了GUI，没看到Smalltalk"用户和开发者无分离"的真正力量，导致后来App框架统治了软件世界
- **产品市场契合度不是"里程碑"而是"渐进坡道"**：没有"达成PMF"的二元时刻，只是"人们开始关心我们制造的东西"、"人们开始付钱给我们"的渐进过程
- **VC开始送狗零食到办公室**：Notion办公室地址未公开，但VC通过Twitter找到办公室地址并给办公室的狗送零食 - "外部验证"的信号
- **"小巴士"隐喻**：巴士越小，越容易转弯、加速、机动；越大越慢。作为领导者，你决定谁坐在你周围的巴士座位上
- **会议室以"永恒工具"命名**：iPhone、原版Macintosh、Lamy 2000钢笔、东芝电饭煲、Sony晶体管收音机 - 这些工具改变了人们的生活并持续了几十年
- **"工匠精神"(Craft)**：将个人价值观和技术技能结合，做出聪明的权衡，创造新的有用事物。妻子称他为"木柜建造者"
- **权衡(Trade-off)的本质**：没有免费午餐，你必须放弃一些东西。关键是放弃"市场/用户在特定时空想要"的"正确"东西
- **AI是"新材料"**：就像铝让大规模航空旅行成为可能、半导体让计算机成为可能一样，语言模型是一种新材料，可以解锁新的权衡方式
- **"系统对抗你"的信号**：以非乐高方式构建时，系统对抗你；以乐高方式构建时，系统为你工作。这是"有机排斥反应"
- **COVID期间的数据库末日时钟**：Notion运行在单个Postgres实例上，扩展到最大实例后面临"完全关闭"的威胁，距离崩溃仅数周时间
- **横向产品的"细分策略"**：大多数人关心"乐高盒子"（现成套装）而非"乐高积木"，需要从"积木思维"转向"解决方案盒子思维"
- **"B2C2B"策略**：消费者个人用户使用Notion做笔记（10亿+用户用例），然后带到工作中。50%的B2B客户来自之前的个人用户
- **"合久必分，分久必合"（《三国演义》）**：纸张时代 → PC时代（dBASE等应用分离） → Microsoft Office套件时代 → SaaS分离时代 → AI时代的重新捆绑
- **AI的三大机会**：
  1. AI写作能力（写作体验）
  2. AI问答连接器（所有信息在一起时搜索和理解更好）
  3. AI编码代理 + 乐高积木（AI擅长组装，可以创建任何垂直用例的定制软件）
- **"我们塑造工具，然后工具塑造我们"（Marshall McLuhan）**：工具延伸我们，然后反过来塑造我们。重要的是你在放大人性的哪个部分
- **模板经济**：非程序员通过Notion模板销售数百万美元，将他们的领域专业知识打包成可销售的工作流

## 3. 心法与原则
- **"高不会太高，低不会太低"**：情绪稳定能力，感觉低落时去睡觉，第二天重置
- **"不要害怕重置"(Don't be afraid to reset)**：通过更好的抽象可以更快地恢复所有进度，因为抽象以指数级而非线性增长
- **"为自己制造，如果足够独特且对他人有用，其他事情会随之而来"**：将事物视为手艺，不断改进
- **"系统比人力更好"**：相信更好的系统比通过人力的蛮力更强大
- **"人才密度"(Talent Density)**：不追踪人数，而是追踪人才密度和人均收入。人们想与更有才华的人一起工作
- **"工艺品与价值观"(Crafts and Values)**：工艺是你的技能和品味，价值观是你的个人价值观和世界观
- **"Lisp内核"哲学**：就像Lisp内核可以非常小（约100行代码）但通过复合产生复杂行为，正确的抽象可以快速追回损失
- **"一对多讲故事"是必要的手艺**：随着公司成长，需要从一对一影响力转变为一对多讲故事能力，这就像游戏中需要解锁的技能

## 4. 实战手册
**坚持长期愿景的5个要素**：
1. **相信这个世界需要这个东西存在**：你需要真正感受"我需要这个成为现实"
2. **保持精简**：必要时裁员到创始人，Notion从5人裁到2人（Ivan和Simon）
3. **不要害怕重置**：重置代码库、心理模型，通过更好的抽象可以指数级恢复
4. **改变环境**：搬到日本等陌生地方，转租公寓和办公室，在日本生活比在旧金山还便宜
5. **日常身心重置**：睡眠重置情绪，可以重置你的代码库和心理模型

**高效运营的实践**：
- **人才密度优先于人数**：追踪"人均收入"而非总人数
- **多技能创始人**：Ivan能设计、做营销、讲故事、达成销售交易，减少对人力的需求
- **用Notion运行Notion**：用产品本身作为元工具构建内部运营系统
- **盈利的正反馈循环**：不需要在18-24个月内筹钱，可以专注于构建
- **办公室设计原则**：
  - 使用家庭式家具而非办公家具
  - 避免"医院式"顶部照明（"顶部照明是邪恶的"）
  - 测试30种暖白色调选择
  - 长期"不穿鞋"政策
  - 围绕一张桌子吃饭
  - 目标是感觉像"艺术家工作室"或"家"

**横向产品战略**：
- **选择大流量用例作为入口**：文档/笔记（10亿+用户）
- **B2C2B模式**：个人用户 → 带到工作环境
- **积木和盒子双轨并行**：
  - 表面：提供解决方案盒子（Lego box）
  - 底层：持续构建乐高积木
- **横向品类扩展**：文档 → 日历 → 邮件（都是10亿+用户用例）

**技术选择教训**：
- **不要过早优化**（Simon的原则） - 但要提前规划
- **避免过早采用新技术**：Web Components当时"太不稳定"，导致必须完全重写
- **单一数据库的危险**：长期运行在单一Postgres实例上，几乎导致崩溃
- **分片数据**：最终解决方案是分片数据库

**产品价值观维护**：
- **识别非乐高构建的信号**：
  - 工程师说"这部分代码太奇怪了"
  - 产品感觉与Notion其他部分不协调
  - 客户困惑"这是什么？"
- **组织排斥反应**：引入反价值观的东西会导致员工和客户的"器官排斥"
- **9-12个月的回归周期**：需要9-12个月意识到偏离价值观并回归

## 5. 反常识视角
- **史蒂夫·乔布斯没有看到PARC的真正价值**：他看到了GUI但没有看到Smalltalk"用户即开发者"的力量，反而批评滚动条方向，演示人员立即改变了它 - 这正是系统力量的体现
- **前3-4年失败是成功的关键**：正是长期失败迫使深入思考"世界不是你"、"人们不在乎"，最终找到"糖衣西兰花"策略
- **盈利状态比融资更自由**：Notion大部分融资仍存在银行，盈利状态让你专注于构建而非筹款
- **VC的"外部验证"是陷阱**：投资人Shana Fisher指出"你显然不需要钱，只是想通过外部验证让自己感觉良好" - 这不能帮助做出更好的产品
- **项目管理第2名就够了**：在6,500人调查中，Notion在项目管理排名第2（Jira之后）、文档第4、CRM第3 - "横向优势"是多种用例在一处
- **"不推荐"横向产品**：Ivan说"我不会推荐别人做"，但"我自己不会做别的选择" - 问题空间太大，没有最佳实践
- **办公室"暖白色"花费数周选择**：测试30种色调不是为了美感，而是因为"如果椅子丑、墙丑，会伤害眼睛"
- **"硬编码"的诱惑和危险**：在项目管理用例中，硬编码Sprint/Milestone概念比创建新的乐高积木更快更容易，但最终"系统对抗你"
- **AI是乐高的完美伴侣**：AI擅长组装，这解决了乐高"太难拼装"的最大弱点 - "谁真正擅长拼装东西？AI"
